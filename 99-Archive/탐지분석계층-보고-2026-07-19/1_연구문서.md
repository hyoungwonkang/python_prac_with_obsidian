# 1. 연구문서 — 탐지 계층 R&D 통합

## 1. 배경과 목적

플랫폼의 **추출 계층**(②: OCR로 이미지→텍스트)과 **1차 탐지 계층**(③: 수집 데이터를 규칙·분류·개체·이미지로
걸러내는 단계)을 공개 데이터로 로컬에서 재현·검증한다. 핵심은 특정 도메인 하나가 아니라 **도메인 교체 가능한 범용 계층** —
모든 모듈이 계약(데이터 규약·프롬프트·라벨 체계)만 바꾸면 새 도메인에 이식되도록 설계했다.
적용처는 아동·청소년 성착취물 대응, 불법스포츠도박 등 복수.

## 2. 모듈별 결과 (전부 로컬 실측)

### 학습 데이터 산출물 환경 (확장성)
과제마다 스크립트를 복사·수정하던 방식을 벗어나 **규약만 맞추면 반복되는 환경**을 구성.
- 순환: 데이터셋(분류 기준의 정본) → 학습 → **산출물 3종 세트**(가중치+label_map+meta) → 재사용 → 검수 → 편입
- 텍스트 범용 학습기(`text,label` CSV만 주면 열·클래스 자동 인식), YOLO 데이터 규약, PII 33라벨 스키마
- 검증: 스팸 스모크 정확도 0.9367 · YOLO 데모 mAP50 0.8718 · PII 33라벨/오타 차단

### 분류 잘하는 법 (Rule vs BERT vs 하이브리드)
고정 test셋 300건에서 동일 잣대 비교. **테스트 누출 금지 3단**(만들 때 train / 조정 validation / 채점만 test).

| 방법 | 정확도 | 스팸P | 스팸R | F1 | 특성 |
|---|---|---|---|---|---|
| RULE (키워드11+패턴4) | 0.900 | 0.948 | 0.848 | 0.895 | 0.12초·설명가능 |
| BERT-full | 0.953 | 0.993 | 0.914 | **0.952** | 균형 최고 |
| HYBRID-AND | 0.913 | **1.000** | 0.828 | 0.906 | 오탐 0 (제재용) |
| HYBRID-OR | 0.940 | 0.946 | **0.934** | 0.940 | 미탐 최소 (회수용) |

**결론: 단일 승자 없음 — 어느 실수(오탐/미탐)가 치명적인가로 방법을 고른다.** 오탐 교집합 0(완전 상보)라 AND에서 오탐 전멸.

### 통합 UXUI 데모
완료 모듈 5개를 **신규 학습 0**으로 조립. Gradio 탭 4개: ① 텍스트 분석(스팸 4종 판정+PII 마스킹+NER) ② 검수·라벨링(순환도 [검수→편입] 실체화) ③ 이미지 분석(YOLO 박스+KoCLIP 상황) ④ 이미지 검색·일괄(유사도 행렬 양방향).

### CLIP 이미지 상황 판단
zero-shot(학습 0줄)→linear probe→KoCLIP 3단계. 같은 test 27장 비교: 부정형 프롬프트 0.444(동전 이하) / 긍정형 0.778 / **KoCLIP+한국어 0.852 최고**. 영어 모델+한국어는 0.519로 붕괴. **분류 체계가 프롬프트에 살아서, 재학습 없이 문장 교체만으로 바뀐다.**

### YOLO 직접 라벨링 → 학습
labelme(로컬 도구)로 직접 라벨링→변환→학습→탐지 실증(mAP50 0.995). **확장 실험**(COCO+신규클래스 81종 병합 재학습): 신규 클래스는 습득했으나 COCO 실력 붕괴(0.177) — **파국적 망각**과 라벨 결손 노이즈 실증. 결론: 소데이터 병합보다 **범용+특화 모델 병행 운용**이 실무 정답.

### OCR 추출 계층 (EasyOCR·PaddleOCR + OpenCV 전처리)
정답을 아는 한국어 문장을 이미지로 렌더링(완벽 조건 + 열화 4종)해 **CER(문자 오류율)** 동일 잣대로 비교.
- 한국어 인식(완벽 조건): **Paddle 4.8% / Easy 9.5%** — 여기서도 단일 승자 없음(정확도 Paddle vs 속도 Easy 5~10배), 오답 프로필 상이(모양 혼동 vs 끝 글자 절단)
- **전처리 = 표적 치료**: 기울기 보정(deskew)이 회전 열화를 완전 복구(64.3→9.5%, 엔진 무관 유효) — 반면 확대는 유령 글자를 만들어 유해(9.5→11.9%). **deskew+Paddle 2.4% 최고**
- **미니 파이프라인(②추출→③탐지)**: 한 글자 오독(`010→0/0`)에 PII 탐지 전멸(4건→0건, 위험도 MEDIUM→INFO — 경보 없는 조용한 실패). 스팸·NER은 중복 단서 덕에 견딤
- 오독 방어선 3층 실측: 전처리(열화 제거) / 신뢰도 게이트(오독 줄이 0.627로 자기 표시) / 두 엔진 교차 검증(불일치만 검수 — 맹점 0, 실사에서 신뢰도 0.89 유령 텍스트 포획)

## 3. 관통하는 발견 (도메인 무관 원리)

- **평가는 두 목록(모델 판정 × 사람 정답)을 맞대는 일** — 정답 없이는 성능이 없다. 라벨링이 곧 자산.
- **닫힌 집합의 억지 배정**: 후보에 없는 입력도 하나로 강제 배정됨 → 캐치올(기타) 클래스와 후보 세트 설계가 필수. 후보 설계 = 오탐 설계.
- **문턱·프롬프트는 조정 변수**: 룰 문턱, CLIP 프롬프트 문구가 성능을 좌우 — validation에서 정한다.
- **의사 라벨링**: 모델 판정은 초안, 사람 확정으로만 정답 승격 (검수 UI로 순환 폐쇄).
- **상류 품질 = 하류 성능**: 뒤 계층은 앞 계층이 준 것만 본다 — 형식 기반 과제(PII)는 한 글자 오독에 무너지고 실패는 조용하다. 계층마다 자기 신뢰도로 자기 문 앞을 지킬 것(뒤 계층 신뢰도는 앞 계층 오염에 침묵).

## 4. 한계와 다음 단계

- 전 모듈이 공개·합성 데이터 기준 — **실전 도메인 데이터에서의 재검·룰 재선정·프롬프트 재확정이 병목** (데이터 확보 필요).
- CLIP 프롬프트 카탈로그는 test에서 고른 부분이 있어 잠정 — 새 데이터로 재확정.
- OCR 수치는 합성 문장 1개·장면 텍스트(coco) 기준 — 실전 도메인 이미지(문서형 캡처)
  파일럿으로 합의율·검수 부담·처리 시간 계수 재측정 필요. 신뢰도 검수 문턱은 라벨 축적 후 validation으로.
