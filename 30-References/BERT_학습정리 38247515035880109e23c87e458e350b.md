## 1. BERT란 무엇인가 — 큰 그림

BERT는 문장을 **한 단어씩 순서대로** 읽는 대신, 문장 전체를 **한꺼번에 펼쳐놓고** 각 단어가 서로 어떤 관계인지 동시에 계산하는 언어 모델입니다.

개발자 비유로 보면, `for` 루프로 배열을 한 칸씩 도는 것과, 배열 전체를 메모리에 올려놓고 모든 원소의 상호 관계를 한 번에 계산하는 것의 차이입니다.

이게 중요한 이유는 같은 철자라도 문맥으로 의미가 갈리기 때문입니다.

- “강가에서 **배**를 탔다” → 보트
- “**배**가 아프다” → 신체
- “**배**가 맛있다” → 과일

뒤에 오는 단어까지 같이 봐야 어떤 ’배’인지 알 수 있는데, BERT는 이를 동시에 보므로 구분이 됩니다. 멀리 떨어진 단어끼리도 직접 연결해서 보기 때문에, 단어 간 거리에 관계없이 중요도를 판단할 수 있습니다.

## 2. 트랜스포머 아키텍처 vs 트랜스퍼 러닝

이름이 비슷해 헷갈리지만 **완전히 다른 개념**입니다. 하나는 ‘구조’, 하나는 ’학습 방법’입니다.

### 트랜스포머 아키텍처 (Transformer Architecture)

모델의 **구조(설계도)** 입니다. “어떻게 생긴 신경망이냐”에 대한 답이며, 핵심 부품이 어텐션(attention)입니다. 개발자 비유로는 React 같은 프레임워크에 해당합니다. BERT는 이 구조로 **만들어진** 모델입니다.

### 트랜스퍼 러닝 (Transfer Learning, 전이 학습)

모델을 **학습시키는 방법(전략)** 입니다. “범용으로 크게 한 번 학습(사전학습) → 내 작업에 맞게 살짝 튜닝(파인튜닝)”하는 2단계 방식으로, 한 곳에서 배운 지식을 다른 작업으로 옮겨(transfer) 재활용합니다. 개발자 비유로는 검증된 base 이미지를 가져와 내 환경에 맞게 커스텀하는 워크플로우입니다.

| 구분 | 트랜스포머 아키텍처 | 트랜스퍼 러닝 |
| --- | --- | --- |
| 정체 | 모델의 **구조** | 모델의 **학습 방법** |
| 답하는 질문 | “어떻게 생겼나?” | “어떻게 훈련하나?” |
| 비유 | React (프레임워크) | base 이미지 재활용 (워크플로우) |

> 한 문장으로: **BERT는 트랜스포머라는 구조로 만들어진 모델을, 트랜스퍼 러닝이라는 방법으로 훈련시킨 결과물**입니다. 둘은 독립적이라 조합이 자유롭습니다.

## 3. BERT 작동 방식 — 6단계 파이프라인

BERT가 텍스트를 받아 처리하는 흐름입니다. 입력 → 전처리 → 핵심 연산 → 출력 순서로 따라가면 됩니다.

### ① 토크나이제이션 — 입력 문자열 쪼개기

모델은 문자열을 직접 못 먹으므로 잘게 쪼개 숫자로 바꿉니다. 쪼개는 단위가 꼭 단어 하나는 아닙니다. `playing`은 `play` + `##ing`로 나뉘는데, `##`은 ’앞 토큰에 붙어있던 조각’이라는 표시입니다.

적은 부품으로 많은 단어를 커버하고, 처음 보는 희귀 단어도 조각으로 쪼개 처리할 수 있습니다. (UTF-8이 적은 바이트 조합으로 모든 문자를 표현하는 발상과 비슷합니다.)

- `[CLS]` — 문장 맨 앞 마커. 문장 전체를 대표하는 자리.
- `[SEP]` — 문장 경계 구분자.

### ② 입력 임베딩 — 숫자에 의미 정보 입히기

토큰 ID만으로는 정보가 빈약하므로 각 토큰에 세 종류의 벡터를 더합니다.

- **토큰 임베딩** → 이 단어가 ’무엇’인지
- **위치 임베딩** → 문장에서 ’몇 번째’인지
- **세그먼트 임베딩** → ‘어느 문장’ 소속인지

위치 임베딩이 핵심입니다. 문장을 동시에 보면 순서 정보가 날아가서 “철수가 영희를 좋아한다”와 “영희가 철수를 좋아한다”가 구분되지 않으므로, 위치 정보를 따로 주입해야 합니다.

### ③ 어텐션 메커니즘 — 진짜 핵심

한 단어를 해석할 때 **다른 어떤 단어를 얼마나 참고할지** 점수를 매깁니다. `bank`를 예로 들면:

- 주변에 `river`, `water` → bank = 강둑
- 주변에 `money`, `deposit` → bank = 은행

```python
# bank의 새 의미 = 모든 단어를 "관련도" 비율로 섞은 것
bank_meaning = (0.6 * river) + (0.3 * water) + (0.05 * the) + ...
```

관련 깊은 단어는 비중 크게, 상관없는 단어는 비중 작게 섞어 문맥이 녹아든 새 표현을 만듭니다.

### ④ 트랜스포머 인코더 레이어 — 여러 겹 쌓기

어텐션 처리를 한 번이 아니라 여러 층 쌓습니다. 한 층의 출력이 다음 층 입력이 됩니다. 미들웨어 체인이나 이미지 필터를 겹쳐 적용하는 것과 비슷합니다.

- **얕은 층** → 기본 문법, 단어 형태 같은 표면적인 것
- **깊은 층** → 추상적 의미, 단어 간 복잡한 관계

CNN에서 앞쪽 층이 선·모서리를, 뒤쪽 층이 얼굴·사물을 잡는 원리와 같습니다.

### ⑤ 사전 학습 작업 — 라벨 없이 스스로 공부

**(1) 마스킹**: 토큰의 15%를 가리고 무엇이었는지 맞히게 합니다.

```
입력:  "나는 아침에 [MASK]를 마셨다"
정답:  커피
```

빈칸을 맞히려면 앞뒤를 다 봐야 하므로 양방향 이해가 길러집니다. 정답이 원문에 이미 있어 사람이 라벨을 달 필요가 없습니다.

**(2) 문장 순서 관계 맞히기**: 두 문장이 실제로 이어지는지 판단해 문장 간 관계를 이해합니다.

> **참고** — 원문에는 “두 번째 문장이 첫 번째보다 앞/뒤에 오는지”로 돼 있으나, 실제 BERT 논문의 작업은 “두 번째 문장이 첫 번째의 실제 다음 문장이 맞는지(연결 vs 무작위)”를 맞히는 것입니다.
> 

### ⑥ 파인튜닝과 추론 — 가져다 쓰기

- **파인튜닝**: 사전학습된 BERT 위에 작업용 얇은 출력 층 하나만 얹어 살짝 더 학습시킵니다. 무거운 본체는 재활용하고 얇은 어댑터만 새로 답니다.
- **추론**: 텍스트가 모든 어텐션 층을 통과하며 문맥이 반영된 표현으로 바뀌고, 마지막 출력 층이 최종 예측을 내놓습니다.

## 4. 마스킹은 왜 필요한가

자주 나오는 의문: “어차피 앞뒤를 다 볼 수 있는데, 왜 굳이 가리고 맞히나?”

핵심은 **’볼 능력이 있다’와 ’학습이 된다’는 별개**라는 점입니다. 마스킹은 능력을 만드는 게 아니라 **학습을 가능하게 만드는 장치**입니다.

### 가리지 않으면 왜 안 되나 — 정답 누출

학습 목표가 “각 자리의 단어를 맞혀라”인데 가리지 않으면, 정답이 입력에 그대로 보입니다.

```
입력:  나는 아침에 커피를 마셨다
문제:  3번째 자리 단어는?
정답:  커피   ← 입력에 이미 있음
```

이러면 모델은 “자기 자리에 적힌 걸 그대로 베껴라”가 돼버립니다.

```python
def predict(word):
    return word   # 입력을 그냥 그대로 반환
```

100점이지만 아무것도 배운 게 없습니다. 컨닝 페이퍼를 쥐고 시험 보는 격이죠. 이를 **information leakage(정답 누출)** 라고 합니다.

### 마스킹이 하는 일

```
입력:  나는 아침에 [MASK]를 마셨다
문제:  [MASK] 자리 단어는?
정답:  커피   ← 입력엔 없음. 베낄 수 없음.
```

이제 베끼기가 불가능하므로 주변 단어를 보고 추론할 수밖에 없습니다. 즉 마스킹은 **베끼는 지름길을 막아 추론이라는 먼 길을 강제하는 장치**입니다.

> **참고** — 같은 ‘정답 누출’ 문제를 GPT는 다르게 풉니다. 가리는 대신 “앞 단어들만 보고 다음 단어 예측”으로 미래를 안 보여주는 방식입니다. BERT는 ‘가리기’, GPT는 ’한쪽만 보기’로 해결한 것입니다.
>

## 5. BERT의 출력은 어떻게 쓰이는가

BERT가 직접 내놓는 건 최종 답이 아니라 **재료**입니다. 문장을 넣으면 각 토큰마다 하나씩, 문맥이 녹아든 **벡터(숫자 배열)** 가 나옵니다.

```
입력:   [CLS] 나는 이 영화 별로 였다 [SEP]
출력:    v0    v1  v2  v3   v4   v5  v6   ← 각각 768개짜리 숫자 배열
```

### 패턴 A — 문장 전체에 대한 답: `[CLS]` 벡터 사용

감성 분석, 스팸 감지처럼 문장 하나에 답 하나를 내는 작업입니다. 맨 앞 `[CLS]` 벡터에 문장 전체 정보가 모이도록 학습돼 있어, 이 벡터 하나가 문장 요약본 역할을 합니다.

```
v0 (문장 대표 벡터)  →  [분류 층]  →  긍정 92% / 부정 8%
```

### 패턴 B — 단어마다 답: 각 토큰 벡터 사용

개체명 인식(NER)처럼 토큰 하나하나에 답을 붙이는 작업입니다. 모든 토큰 벡터를 각각 분류 층에 통과시킵니다.

```
v1(애플)     → 회사
v2(이)       → 없음
v3(쿠퍼티노) → 지명
```

| 단계 | 하는 일 | 비유 |
| --- | --- | --- |
| BERT 본체 | 문맥 녹인 벡터 생성 | 식재료를 손질해서 내놓음 |
| 작업별 층 | 벡터 → 실제 답 | 손질된 재료로 요리 완성 |

핵심은 **역할 분리**입니다. 본체는 공유하고 머리(작업별 층)만 갈아 끼우면 감성 분석·NER·QA용을 다 만들 수 있습니다. 이 구조 때문에 BERT는 분류·태깅·선별은 잘하지만, GPT처럼 새 문장을 써내려가는 생성은 못 합니다.

## 6. NER과 자연어 라벨링

### NER은 BERT가 곧 본체

NER은 별도 시스템이 BERT 결과를 받아 가는 구조가 아니라, **BERT + 그 위 분류 층**으로 이루어진 하나의 모델입니다.

```
[BERT 본체] + [NER용 분류 층]  =  NER 모델  (한 덩어리)
```

> **참고** — ’에이전트’는 보통 LLM이 스스로 판단해 도구를 호출하고 여러 단계를 밟는 시스템을 가리킵니다. NER 모델은 입력을 넣으면 결과가 나오는 단순 함수에 가까워, 스스로 행동하는 에이전트는 아닙니다.
> 

또한 **BERT는 텍스트 전용**이라 이미지는 입력으로 받지 못합니다. 이미지는 CNN이나 ViT(Vision Transformer)가 담당하며, 멀티모달 모델에서만 BERT 계열이 텍스트 파트를 맡아 이미지 파트와 결과를 합칩니다.

### ’라벨링 = NER’은 아니다

라벨링이라는 우산 아래 여러 도구가 있고, 목적에 맞는 걸 골라야 합니다. NER은 그중 ‘문장 속 개체에 태그 다는’ 특정 작업만 담당합니다.

| 하고 싶은 일 | 맞는 기법 | NER로? |
| --- | --- | --- |
| 단어에 개체 태그 (사람/지명/회사) | **NER** | ⭕ |
| 문장 전체 라벨 (긍정/부정, 스팸) | 텍스트 분류 | ❌ |
| 단어마다 품사 태그 (명사/동사) | POS 태깅 | ❌ |
| 문장 주제·의도 분류 | 텍스트 분류 | ❌ |
| 단어들 사이 관계 뽑기 | 관계 추출 | ❌ |

> **참고** — 학습용 데이터에 정답을 붙이는 ’라벨링(annotation)’을 NER로 자동화하려는 경우, 순서가 거꾸로입니다. NER 모델 자체가 사람이 라벨링한 데이터의 결과물이기 때문입니다. 다만 실무에선 학습된 NER로 초벌 라벨링을 돌리고 사람이 검수·수정하는 식으로 일을 덜어냅니다.
>

## 7. 핵심 요약

전체를 한 줄로 꿰면:

> 문자열을 토큰으로 쪼개고(①) → 숫자에 의미·위치·소속 정보를 입히고(②) → 어텐션으로 단어 간 관계를 계산하길(③) 여러 층 반복해(④) 문맥을 녹여낸 표현을 만든 뒤, 빈칸 맞히기로 미리 언어를 익혀두고(⑤) 내 작업용 층만 얹어 가져다 쓴다(⑥).
>
