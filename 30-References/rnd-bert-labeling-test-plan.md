# R&D 산출물 계획 — BERT 모델 · 라벨링 방법론 · 테스트

> **업무 요청 대응 계획.** R&D 주제는 **① BERT 모델 ② 라벨링 방법론 ③ 테스트** 세 가지(동등).
> 이 세 주제를 한 번에 실증할 **구체 사례(vehicle) = SMS 스팸분류**를 사용 — 분류 하나로 BERT 모델·라벨링·테스트가 모두 드러남.
> 참고 정본: BERT 개념 [[BERT_학습정리]]·[[bert_ocr_practice_plan]], 라벨링/분할 실습 [[../10-Projects/llm-from-scratch/llm-ch6-classify]], 환경 [[pytorch-env-hybrid]].

## 0. 요청 내용 (업무 메시지)

> "금일(연구문서/소스/사용법/가이드) **또는** 화요일(시연)까지
> **BERT 모델 및 라벨링 방법론 및 테스트** R&D 내용을 공유."

| 경로 | 마감 | 내야 하는 것 |
|---|---|---|
| **A (문서 경로)** | **오늘 (2026-06-26 금)** | **4종 전부** — 연구문서 / 소스 / 사용법 / 가이드 |
| **B (시연 경로)** | 화요일 (2026-06-30) | 시연(데모) |

- `또는` = **두 개의 완결 경로 중 택1** (오늘 문서 4종 / 화요일 데모). "오늘 일부 + 화요일 나머지"로 섞는 게 아님.
- **선택: A를 오늘 완수**(원래 업무) → **B는 화요일 시연으로 03·04 검증·실증** (보너스/완성).

## 1. 핵심 구조 — 4종은 "형식", 3주제는 "내용"

4개 산출물은 서로 다른 주제가 아니라, **같은 3주제(BERT 모델·라벨링 방법론·테스트)를 4가지 형식으로** 푸는 것.

```
내용(R&D):  BERT 모델  +  라벨링 방법론  +  테스트
                    │
      ┌─────────────┼─────────────┬─────────────┐
   연구문서       소스코드        사용법         가이드   (형식 4종)
```

### 산출물 × 주제 매트릭스 (모든 칸을 채워야 완성)

| 산출물(형식) | BERT 모델 | 라벨링 방법론 | 테스트 |
|---|---|---|---|
| **01 연구문서** | 구조·어텐션·`[CLS]`분류 (기존 [[BERT_학습정리]]) | 라벨링 개념 + 스팸 실무(ham/spam→0/1·균형·70:10:20) | 테스트 전략(test셋·정확도·혼동행렬) |
| **02 소스코드** | `BertForSequenceClassification` 로드·학습 | 데이터 라벨링 코드 | 평가 코드(accuracy·confusion_matrix) |
| **03 사용법** | 모델 학습 실행 명령 | 데이터 준비 실행 | 평가 실행·결과 보는 법 |
| **04 가이드** | 전체 재현 절차 | 라벨링 재현 | 테스트 재현·주의점(Colab GPU) |

> 핵심 구분: **"소스코드 산출물 = 작성된 코드 문서"** 이지 *돌아가는 데모(B)*가 아님.
> → 로컬에 transformers 없어도, Colab 실행 전이어도 **오늘 4종 작성 가능**. 단 03·04는 **실행 전이라 미검증** → 화요일 시연이 그 검증.

## 4. 기술 스택

| 항목 | 사용할 것 |
|---|---|
| 모델 | HF `BertForSequenceClassification` (`bert-base-uncased`, `num_labels=2`) |
| 토크나이저 | `BertTokenizer.from_pretrained` |
| 데이터 | SMS Spam Collection (라벨링·분할 완료된 CSV 재사용) |
| 평가 | accuracy + confusion matrix (sklearn) |
| 실행 환경 | **Colab GPU** (로컬은 작성만, 실행은 Colab) → [[pytorch-env-hybrid]] |

## 5. 진행 순서

1. **오늘 (A 경로 = 원래 업무) — 4종 md 전부 작성**
   - `01-연구문서.md` → `02-소스코드.md` + `finetune_bert_spam.py` → `03-사용법.md` → `04-가이드.md`
   - 작업 팁: **02 코드 먼저** 확정해야 03·04(사용법·가이드)가 정확.
2. **화요일 (B 경로) — 시연·검증**
   - `finetune_bert_spam.py`를 **Colab GPU에서 실제 실행** → 03·04를 실행 결과로 확정·정정, 데모.
