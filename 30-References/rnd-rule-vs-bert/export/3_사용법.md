# 3. 사용법 — 분류 방법 비교 (2026-07-09)

> 실행 환경: `~/rnd-env` (Python 3.12). 모든 명령은 `export/` 폴더 안에서 실행 기준.

## 1) 룰 문턱 확인 (validation 기준 — 근거 재현)

```bash
~/rnd-env/bin/python rule_spam.py
# → 문턱 1~3의 정확도/P/R/F1 출력 (test는 열지 않음)
```

## 2) 방법 비교 채점 (고정 test셋 300건)

```bash
# 기본: RULE + BERT-smoke
~/rnd-env/bin/python eval_compare.py

# BERT 산출물 지정 (콤마 구분 — 3종 세트 폴더면 무엇이든)
ARTIFACTS=../../rnd-dataset-artifacts/export/artifacts/ko-spam-smoke,../../rnd-dataset-artifacts/export/artifacts/ko-spam-full \
  ~/rnd-env/bin/python eval_compare.py
```

| 파라미터 | 기본값 | 설명 |
|---|---|---|
| `TEST` | ko 스팸 test.csv | 고정 test셋 경로 (바꾸면 잣대가 바뀌므로 신중히) |
| `ARTIFACTS` | ko-spam-smoke | 채점할 BERT 산출물 폴더들 (콤마 구분) |

## 3) 오답 정성 분석 + 하이브리드

```bash
~/rnd-env/bin/python analyze_errors.py
# → 미탐·오탐의 교집합/차집합, 그룹별 오답 문장, HYBRID-OR/AND 성적
```

| 파라미터 | 기본값 | 설명 |
|---|---|---|
| `ARTIFACT` | ko-spam-full | 하이브리드에 쓸 BERT 산출물 1개 |

## 4) 새 방법을 비교에 추가하려면

1. **BERT 계열**: 지시 1 학습기로 학습 → 산출물 3종 세트가 생기면 `ARTIFACTS=`에 폴더 추가. 끝.
2. **룰 개선판**: rule_spam.py의 KEYWORDS/PATTERNS 수정 — 단, **근거는 train·validation에서만**
   (test 오답을 보고 고치면 누출).

## 5) 실험 기록 확인 (MLflow)

```bash
~/rnd-env/bin/mlflow ui --backend-store-uri sqlite:///mlflow.db --port 5001
# → http://localhost:5001 (실험: 분류방법-비교 — RULE·BERT들·HYBRID run 비교)
```
