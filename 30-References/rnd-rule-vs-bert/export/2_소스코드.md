# 2. 소스코드 — 분류 방법 비교 (2026-07-09)

파일 3개. 실행 환경은 `~/rnd-env` (Python 3.12).

## rule_spam.py — 룰 분류기 (베이스라인)

```
KEYWORDS 11개  ← train 통계(스팸 출현 ≥8 · 정상 출현 0)에서 수작업 선별. 각 항목에 근거 주석
PATTERNS 4개   ← 5자리+ 숫자 / £ / 미번역 영어 스팸 어휘 / URL
score(text)    ← 맞은 신호 개수
classify(text) ← score ≥ THRESHOLD(1) 이면 스팸
__main__       ← validation 149건으로 문턱 1~3 채점 (문턱 결정 근거를 재현 가능하게)
```

설계 결정:
- **누출 금지가 코드 구조에 반영** — 키워드 선정(train)·문턱 조정(validation)·채점(test)의
  3단 분리. 문턱 조정 루틴이 `__main__`에 있어 근거가 스크립트로 남는다.
- 키워드마다 "스팸출현/정상출현" 주석 — 왜 이 단어인지 추적 가능 (룰의 설명가능성).

## eval_compare.py — 동일 잣대 채점기

```
four_cells()   ← 혼동행렬 네 칸 (정탐 tp·오탐 fp·미탐 fn·tn) — p==1(판정)과 p==g(채점)의 조합
eval_bert()    ← 산출물 3종 세트만으로 분류기 재조립 (meta→뼈대, model.pt→가중치, label_map→번역)
report()       ← 지표 계산 + MLflow run 기록 + 한 줄 출력
main()         ← RULE + ARTIFACTS(콤마 구분 폴더들)를 같은 test셋으로 순회 채점
```

설계 결정:
- **학습 코드 불의존** — train_text.py를 import하지 않음. 3종 세트 규약만 지키면
  미래의 어떤 모델도 `ARTIFACTS=폴더` 지정만으로 이 비교 잣대에 올라온다.
- MLflow는 sqlite 명시 고정 (지시 1 R&D에서 확인한 3.x 파일 저장소 함정 회피).

## analyze_errors.py — 오답 정성 분석 + 하이브리드

```
미탐·오탐을 집합으로 수집 → 교집합/차집합 계산 (정량: 칸막이 세우기)
show()로 그룹별 문장 출력 (정성 분석의 재료 — 분석 자체는 사람이 읽고 병명 부여)
HYBRID-OR / HYBRID-AND 채점 → MLflow 기록
```

설계 결정:
- 하이브리드는 예측 결합만으로 구현 (재학습 없음) — `r or b` / `r and b` 각 한 줄.
- 그룹 구분(둘 다/RULE만/BERT만)이 정성 분석의 읽기 순서를 정해준다.
