# 4. 가이드 — "분류 잘하는 법"을 판단하는 틀 (2026-07-09)

이 문서는 코드가 아니라 **판단 기준**을 남긴다. 새 분류 과제를 만났을 때 이 순서로 생각한다.

## 1. 먼저 물을 것: 어느 실수가 치명적인가

분류기의 실수는 두 종류뿐이다 — **오탐**(정상을 건드림, P를 깎음) / **미탐**(이상을 놓침, R을 깎음).
업무가 어느 쪽에 민감한지가 방법 선택의 출발점이다.

| 업무 성격 | 치명적 실수 | 권장 (이번 실측 기준) |
|---|---|---|
| 판정 뒤 차단·제재가 자동 실행 | 오탐 | HYBRID-AND (P 1.0000, 오탐 0) |
| 놓치면 사고 (수사 단서·안전) | 미탐 | HYBRID-OR (R 0.9338, 미탐 하한) |
| 사람 검수가 뒤에 있음, 균형 | 둘 다 적당히 | BERT 단독 (F1 0.9517) |
| 서버 없음·실시간·근거 제시 의무 | (성능 약간 양보) | RULE 단독 (0.12초, 설명 가능) |

## 2. 항상 지킬 것: 테스트 누출 금지 3단

```
만들 때는 train  →  조정할 때는 validation  →  채점할 때만 test
```

- 룰의 키워드든 모델의 하이퍼파라미터든, **test를 보고 정하면 그 순간 비교가 무효**.
- 오답 정성 분석은 test를 읽는 작업이므로 **용도는 이해·보고까지** — 발견한 개선 아이디어는
  train/validation에서 같은 패턴을 재확인한 뒤에만 반영한다.

## 3. 앙상블을 검토할 때: 오답의 교집합부터 잰다

- 결합의 이득은 **오답이 얼마나 안 겹치는가**가 결정한다:
  **AND의 오탐 = 오탐 교집합 / OR의 미탐 = 미탐 교집합** (집합 항등식 = 성적의 상·하한).
- 이번 실측: 오탐 교집합 0 (완전 상보) → AND가 오탐을 전멸시킴.
  미탐 교집합 10 → OR로도 10건은 못 구함 (이건 데이터·방법 공통의 한계).
- 오답이 거의 겹친다면 앙상블 이득이 없다 — 그때는 결합 대신 더 나은 단일 방법을 찾는다.

## 4. 수치가 나온 다음: 정성 분석으로 병명을 붙인다

수치는 "어디에 서 있는가", 정성 분석은 "다음에 뭘 할 것인가". 오답을 직접 읽고 원인을 분류한다:

- **문맥맹** (단어는 맞는데 문맥이 다름 — "파운드 빠졌다"=체중) → 룰로 못 고침, 모델의 몫
- **패턴 구멍** (규칙이 못 덮는 표기 변형 — 띄어쓰기 전화번호) → 규칙 보완으로 해결 가능
- **라벨 애매** (사람이 봐도 판단 곤란) → 방법이 아니라 데이터 검수의 몫

병명마다 처방이 다르므로, 병명 없이 "성능이 낮으니 더 학습"은 무작위 시도다.

## 5. 룰을 얕보지 말 것

- train 통계에서 뽑은 키워드 11개로 F1 0.895 — **강한 베이스라인은 공짜에 가깝다.**
  새 과제는 항상 룰 베이스라인부터 만들어 "모델이 이만큼 더 나은가"를 잴 것.
- 룰의 진짜 가치는 점수 밖에 있다: 속도(0.12초), 설명가능성(어느 키워드에 걸렸는지),
  무서버, 그리고 앙상블의 부품 (BERT의 오탐에 거부권을 행사).

## 6. 이 실험 자체의 재사용법

고정 test셋 + 산출물 3종 세트 규약 + eval_compare.py 조합은 스팸 전용이 아니다.
새 도메인(도박 권유 등)에서도: ① 3분할 데이터 준비 ② 룰 베이스라인 제작(train만) ③ 지시 1
학습기로 BERT 학습 ④ eval_compare로 동일 잣대 채점 — 같은 4단계가 그대로 반복된다.
