# 1. 연구문서 — 분류 방법 비교: Rule vs BERT vs 하이브리드 (2026-07-09)

## 1. 배경과 목적

상사 지시 2번 "분류 잘하는 법"에 대한 실측 답변. 좋은 분류란 지표 하나가 아니라
**업무 요구(어느 실수가 치명적인가)에 맞는 방법을 고르는 것**이라는 가설을,
같은 잣대(고정 test셋) 위에서 여러 방법을 비교해 검증한다.

학습 데이터 산출물 환경(지시 1)의 첫 실전 적용이기도 하다 — 고정 test셋·산출물 3종 세트·
MLflow 비교 관례가 그대로 이 실험의 인프라가 됐다.

## 2. 실험 설계

- **고정 test셋**: 한국어 스팸 300건 (스팸 151 / 정상 149) — 모든 방법을 같은 300건으로 채점
- **테스트 누출 금지 3단 배치**:
  1. 룰 키워드 = train(1,045건) 통계로만 선정 (스팸 출현 ≥8 · 정상 출현 0)
  2. 룰 문턱 = validation(149건)으로 조정 (문턱 1: F1 0.9487로 채택)
  3. test = 최종 채점에만 사용 (규칙 제작 중 열지 않음)
- **기록**: MLflow sqlite, 실험 `분류방법-비교` — 방법 1개 = run 1개

## 3. 비교한 방법 5종

| 방법 | 내용 |
|---|---|
| RULE | 키워드 11개 + 정규식 패턴 4개, 문턱 1 (신호 1개면 스팸) |
| BERT-smoke | 지시 1 환경의 기존 산출물 (train 300건·1에폭) |
| BERT-full | 지시 1 학습기 재사용, train 1,045건·2에폭 — **코드 수정 0줄, 학습 25초** |
| HYBRID-OR | 룰 또는 BERT가 스팸이라 하면 스팸 (한 표제) |
| HYBRID-AND | 둘 다 스팸이라 할 때만 스팸 (만장일치제) |

## 5. 오답 분석 — 두 방법은 다른 곳에서 틀린다

**교집합 구조** (하이브리드 성적을 결정한 실측):

- 미탐: RULE 23 / BERT-full 13 / **교집합 10** → 상보 16건 = 앙상블의 여지
- 오탐: RULE 7 / BERT-full 1 / **교집합 0** → 완전 상보 = AND에서 오탐 전멸

집합 항등식이 그대로 성적이 됨: **OR의 미탐 = 미탐 교집합(10)**, **AND의 오탐 = 오탐 교집합(0)**.

**오답의 병명 3종** (정성 분석 — 원인별 처방이 다름):

| 병명 | 사례 | 처방 |
|---|---|---|
| 문맥맹 (룰 오탐 주원인) | "4 **파운드**를 잃었다"(체중), "**휴가**가 없다"(일상) | 룰로는 불가 — BERT의 몫 |
| 패턴 구멍 (룰 미탐) | 띄어쓰기 전화번호 "0800 542 0578", 맨 도메인 | 정규식 보완 가능 — 단 train/val에서 재도출 |
| 라벨 애매 (공통 미탐 10건 일부) | 번역이 뭉개져 사람도 판단 곤란 | 방법이 아니라 데이터 검수 |

## 6. 결론 — 단일 승자 없음, 업무 요구별 선택지 메뉴

| 업무 요구 | 권장 | 근거 |
|---|---|---|
| 오탐 절대 불가 (차단·제재 후속) | **HYBRID-AND** | P 1.0000, 오탐 0 |
| 미탐 최소 (회수가 생명) | **HYBRID-OR** | R 0.9338, 미탐 10(하한) |
| 균형 한 숫자 | **BERT-full 단독** | F1 0.9517 |
| 속도·설명가능성·무서버 | **RULE 단독** | 0.12초, 근거 제시 가능, F1 0.895 |

부수 확인: 룰도 "데이터에서 배운" 것(train 통계로 선정) — BERT와의 차이는 배움의 형태
(읽을 수 있는 조건 11개 vs 읽을 수 없는 가중치 1.1억 개)다.

## 7. 한계와 다음 단계

- **잣대 불일치**: 키워드 선정은 공백 토큰 세기, 판정은 부분 문자열 매칭 — "현금"이 토큰으론 34/0,
  부분 문자열론 44/2. 오탐 일부의 원인 추정 → 부분 문자열 기준으로 재선정이 개선 후보
- **한 변수 원칙 위반**: smoke→full은 데이터량·에폭을 동시 변경 — 기여 분리에는 1,045건·1에폭 중간점 필요
- 에폭 2에서 검증정확도 상승 중(0.9597→0.9664) — 3에폭 시험 여지
- 오답 정성 분석은 test를 본 것 — **여기서 나온 개선 아이디어는 train/val에서 재도출해야 적용 가능** (누출 방지)
- 공통 미탐 10건의 라벨 재검수 (데이터 품질의 몫)
