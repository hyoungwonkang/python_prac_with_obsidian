# 00 · 학습메모 — 분류 방법 비교 R&D에서 익힌 개념 (2026-07-09)

> 지시 2 진행 중 Q&A로 체득한 **새 개념만** 누적 (보고물 아님 — 01~04와 분리).
> 선행 개념은 반복하지 않음: P·R 분모·두 원·판정 어휘·두 가족은 [[../ml-eval-training-terms]](같은 날 오전),
> 오탐/미탐/정탐·트레이드오프는 [[../mlflow-practice/mlflow-terms-glossary]] §5·§7.

## 실험 방법론

### 1. 테스트 누출 금지의 3단 배치
- **만들 때 train → 조정할 때 validation → 채점할 때만 test.** 룰의 키워드·문턱도 모델의
  하이퍼파라미터와 똑같이 이 규율을 따른다 — test를 보고 정하는 순간 비교 무효.
- 오답 정성 분석은 test를 읽는 작업 → **용도는 이해·보고까지.** 개선 아이디어는 train/val에서
  같은 패턴을 재확인한 뒤에만 반영.

### 2. 문턱(threshold) = P·R 트레이드오프의 손잡이 (실측)
- 같은 룰인데 문턱 1→2: P 0.961→**1.000** / R 0.937→0.709. 요구사항에 따라 같은 부품을
  다르게 세팅할 수 있다 — "방법"만이 아니라 "설정"도 선택지.

### 3. "한 번에 한 변수" 원칙
- smoke(300건·1에폭)→full(1,045건·2에폭)은 두 변수 동시 변경 — 향상의 원인(데이터? 에폭?)을
  분리할 수 없다. 기여를 가리려면 중간점(1,045건·1에폭)이 필요.
- 에폭 수는 validation이 정한다: 검증정확도 0.9597→0.9664 **상승 중** = 2에폭은 아직 미과적합
  (꺾이는 지점이 정지 신호).

### 4. 정성 분석은 방법이다 — 표현과의 구분
- 판별 기준: **데이터에 없던 지식이 새로 생겼는가.** 오답 문장 출력은 표현(재료 제시),
  읽고 원인 라벨(병명)을 붙여 범주화하는 것이 분석 — 질적 연구의 코딩(coding) 약식.
- 병명 3종과 처방: **문맥맹**(파운드=체중 — 룰로 불가, 모델의 몫) / **패턴 구멍**(띄어쓰기
  전화번호 — 정규식 보완 가능) / **라벨 애매**(뭉개진 번역 — 데이터 검수의 몫). 병명 없이
  "성능 낮으니 더 학습"은 무작위 시도.
- 프로세스에서의 자리 = **수치 샌드위치**: 채점(정량) → 병명(정성) → 가설 → 재채점(정량).
  수치는 위치를, 정성은 다음 행동을 알려준다.

## 룰과 데이터

### 5. 룰도 데이터에서 배운다 — 차이는 배움의 형태
- 키워드는 train 통계(스팸 출현/정상 출현)에서 선정 — 룰 제작도 일종의 학습.
  BERT와의 차이는 **읽을 수 있는 조건 11개 vs 읽을 수 없는 가중치 1.1억 개**.
- 선정 기준 2열이 곧 단어 수준의 P·R 설계: **정상 출현 = 미래의 오탐 후보(P 보호),
  스팸 출현 = 커버리지(R 확보)**. "무료"(89/9)처럼 스팸에 흔해도 정상에 나오면 탈락.

### 6. 잣대 불일치 함정 (실측으로 발견)
- 키워드 **선정**은 공백 토큰 세기("현금" 34/0), **판정**은 부분 문자열 매칭("현금" in text →
  44/2 — "현금으로" 안에서도 발화). 통계의 잣대와 룰의 잣대가 다르면 "정상 출현 0" 보증이 깨진다.
  → 개선: 부분 문자열 기준으로 재선정.

## 지표와 앙상블

### 7. FP↓ ⟺ TN↑ 항등식 — 네 칸의 행 구조
- 실제 정상은 두 칸뿐(FP+TN=고정), 실제 이상도 두 칸뿐(TP+FN=고정) → 네 칸의 실질 자유도는 2.
- **특이도(specificity) = TN/실제 정상** — "정상 보호율"의 표준 이름 (민감도=재현율의 짝).
- FP 하나가 줄면 **두 지표가 동시에** 좋아진다: 특이도(행 방향)와 정밀도(열 방향).

### 8. P·R 문장의 "두 가족 한 번씩" 문법
- 올바른 지표 문장에는 **판정 가족 단어 1회 + 정답 가족 단어 1회**가 순서만 바꿔 들어간다:
  P = "스팸이라 **판단한** 것 중 **진짜** 스팸 비율" / R = "**실제** 스팸 중 스팸이라 **판단한** 비율".
- 함정 목록(전부 직접 밟음): 틀린 조각을 지표로 / 조각을 분모로 / 특이도 정의를 P에 /
  출발 명단이 실제면 R 문장 / 판단이 두 번 나오면 순환 / 이중부정.
- 한국어는 목적어가 앞에 오므로 "스팸문자를 ~하는"으로 시작하면 자동으로 R로 흘러간다 —
  P를 말하려면 "판단한 것 중"이 먼저.

### 9. 앙상블의 집합 산수 — 오답 교집합이 성적을 결정
- **AND의 오탐 = 오탐 교집합 / OR의 미탐 = 미탐 교집합** (항등식 = 결합 성적의 상·하한).
- 실측: 오탐 교집합 0(완전 상보) → AND에서 오탐 전멸(P 1.0). 미탐 교집합 10 → OR의 하한.
- 구조 비유: AND = 만장일치제(서로의 단독 판정에 거부권), OR = 한 표제.
  오답이 거의 겹치면 앙상블 이득 없음 — 결합 전에 교집합부터 잴 것.

### 10. "분류 잘하는 법"의 답 = 선택지 메뉴
- 단일 승자 없음. 오탐 절대 불가→AND(P 1.0) / 회수 생명→OR(R 0.934) / 균형→BERT 단독(F1 0.952)
  / 무서버·설명 의무→RULE(0.12초). **좋은 분류 = 업무의 치명적 실수에 방법을 맞추는 것.**
