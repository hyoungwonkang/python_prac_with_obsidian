# LLM/ML 개념 용어집

> 학습 중 정리하는 **개념·구조 용어** (ch1 체크리스트 1.5에서 예고). 학습 트랙: [[../10-Projects/llm-from-scratch]].
> PyTorch API 용어(tensor·cross_entropy·AdamW 등)는 → [[python-basics]] "PyTorch 기초". MLOps/MLflow는 → [[mlops]].

## AI ⊃ ML ⊃ DL (포함 관계)
```
AI (지능적 작업 전반, 규칙기반 포함)
 └ 머신러닝 ML (데이터에서 패턴/규칙을 스스로 학습)
     └ 딥러닝 DL (여러 층 신경망, 특징도 자동 추출)
         └ 트랜스포머 / LLM (GPT)
```
| 층위 | 정의 | 예 |
|---|---|---|
| AI | 사람 같은 지능적 작업 | 체스 엔진, 전문가 시스템 |
| **ML** | 데이터로 패턴 학습 | sklearn 로지스틱 회귀(Iris) |
| **DL** | 여러 층 신경망 학습 | CNN, 트랜스포머, GPT |

- **고전 ML vs 딥러닝 핵심 차이 = "특징(feature)을 누가 만드나"**:
  - 고전 ML(Iris): 사람이 특징을 정해 줌(꽃받침 길이·너비 4개).
  - 딥러닝(GPT): 특징 추출까지 모델이 자동 → 이게 **백본**의 역할.
- MLflow는 ML **전반**(고전 ML·딥러닝)을 관리 → 그래서 Iris로 배워 GPT에 적용 가능했음.

## 파운데이션 모델 (foundation model)
- **정의**: 방대한 데이터로 사전학습돼 **여러 작업의 토대**가 되는 대형 모델 (예: GPT-2). 2021 Stanford 제안 용어, 언어 외 이미지(CLIP)·멀티모달 포함.
- **교재(Raschka) 틀**: **5장(사전훈련)의 결과물 = 파운데이션 모델.** 그걸 확보하는 활동 = ① 훈련 반복 ② 모델 평가 ③ 사전훈련 가중치 로드. → 6·7장에서 파인튜닝.
  - 즉 "활동 ≠ 파운데이션 모델", "활동으로 파운데이션 모델을 *확보*" (모델은 결과물).
- **실용**: 처음부터 사전훈련은 비쌈(파라미터×토큰×에포크) → **공개 가중치(GPT-2) 로드로 싸게 확보**(5.5).
- 비유: 대학까지 마친 사람(범용 지식, 교육 비쌈) → 파인튜닝 = 특정 직무 단기 연수.

## 백본 (backbone) / head
- **백본**: 입력을 **유용한 표현(특징)으로 바꾸는 핵심 본체**(head 제외 몸통). 등뼈처럼 공통으로 떠받침.
  - GPT 백본 = 임베딩 + 트랜스포머 블록 12개.
- **head**: 백본 출력 위에 붙는 **작업 전용 얇은 층**. GPT 생성 head = `out_head`(768→50257). 분류는 분류 head로 **교체**(6장).
- **파인튜닝** = 백본 재사용 + head 교체(또는 백본 일부만 미세조정).
- **파운데이션 모델 vs 백본**: 파운데이션 = 전체 기반 모델(백본+head), 백본 = 그 안 특징추출 본체.
- 비유: 백본=자동차 섀시/엔진(공통), head=용도별 적재함(트럭·밴·버스).
- 분야: 비전 CNN(ResNet/VGG), 언어(GPT/BERT 트랜스포머 블록) — "특징 추출 본체"의 범용어.
