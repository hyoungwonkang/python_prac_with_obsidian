# 3. 사용법 — 학습 데이터 산출물 환경 (2026-07-08)

> 실행 환경: Python 3.12 가상환경 (`~/rnd-env`). 없다면:
> ```bash
> python3.12 -m venv ~/rnd-env
> ~/rnd-env/bin/pip install -r requirements.txt
> ```
> 모든 명령은 `export/` 폴더 안에서 실행 기준.

## 1) 텍스트 분류 — 새 데이터로 분류기 만들기

### 데이터 준비 (입력 계약)

둘 중 하나:
- **CSV 1개** — `text`·`label` 이름이 들어간 두 열 (대소문자 무관, 라벨은 문자열 권장).
  70:10:20으로 자동 분할된다.
- **디렉터리** — `train.csv` / `validation.csv` / `test.csv` 3개 파일.

### 학습 → 산출물 생성

```bash
DATA=/path/to/my_data.csv NAME=my-classifier ~/rnd-env/bin/python train_text.py
```

| 파라미터 | 기본값 | 설명 |
|---|---|---|
| `DATA` | (ko 스팸 디렉터리) | CSV 파일 또는 3분할 디렉터리 |
| `NAME` | 데이터 이름 | 산출물 폴더명 (`artifacts/<NAME>/`) |
| `BASE` | `klue/bert-base` | 베이스 모델 (한국어 기본) |
| `EPOCHS` / `BATCH` / `LR` / `MAX_LEN` | 1 / 16 / 2e-5 / 128 | 학습 하이퍼파라미터 |
| `LIMIT` | 0 (전체) | 분할별 상한 — 스모크 테스트용 |
| `EXPERIMENT` | 텍스트분류-범용 | MLflow 실험명 |

### 산출물로 새 텍스트 분류 (재사용)

```bash
ARTIFACT=artifacts/my-classifier ~/rnd-env/bin/python classify_text.py "분류할 문장"
```

## 2) YOLO — 직접 라벨링한 데이터로 탐지기 만들기

### 데이터셋 뼈대 생성 → 라벨링해서 채우기

```bash
NAME=my-dataset CLASSES=사람,차량 ~/rnd-env/bin/python make_yolo_dataset.py
# → datasets-own/my-dataset/ 생성 (images/·labels/·data.yaml·README)
# labelImg·Roboflow·CVAT에서 "YOLO 형식"으로 내보내 images/·labels/에 배치
```

### 규약 검증용 데모 (라벨링 없이 end-to-end 확인)

```bash
DEMO=1 ~/rnd-env/bin/python make_yolo_dataset.py     # 합성 원/사각형 데이터 생성
DATA=datasets/yolo-demo/data.yaml ~/rnd-env/bin/python yolo_train_custom.py
```

### 커스텀 학습 → 산출물 생성

```bash
DATA=datasets-own/my-dataset/data.yaml EPOCHS=40 ~/rnd-env/bin/python yolo_train_custom.py
```

| 파라미터 | 기본값 | 설명 |
|---|---|---|
| `DATA` | 데모 data.yaml | 데이터셋 정본 선언 파일 |
| `EPOCHS` / `IMGSZ` | 40 / 320 | 소량 데이터 기준값 |
| `BASE` | `yolov8n.pt` | 사전학습 가중치 (최초 실행 시 다운로드) |

## 3) PII — 분류 체계 문서와 라벨 검증

```bash
~/rnd-env/bin/python make_pii_schema.py        # pii_label_schema.md/json 생성·갱신
~/rnd-env/bin/python validate_pii_samples.py   # 평가 데이터 라벨이 33라벨 체계인지 검사
# 통과=종료코드 0, 위반=1 (위반 줄과 라벨을 출력)
# 다른 샘플 파일 검사: SAMPLES=/path/to/samples.txt 를 앞에 붙임
```

평가 데이터 형식: `문장<TAB>기대라벨1,기대라벨2` (라벨은 `pii_label_schema.md`의 33개 중에서).

## 4) 산출물 저장 규칙

```
export/
  artifacts/<이름>/          ← 산출물 3종 세트 (git 제외 — 재학습으로 재생성)
    model.pt|best.pt         가중치
    label_map.json           숫자↔클래스명 (재사용의 열쇠)
    meta.json                베이스모델·지표·MLflow run id
  datasets/                  ← 공개·합성 데이터 (git 제외 — 재생성 가능)
  datasets-own/              ← 직접 라벨링한 데이터 (git 추적 — 재생성 불가한 자산)
  mlflow.db                  ← 실험 기록 sqlite (git 제외)
```

**규칙**: 직접 만든 라벨 데이터만 git에 남긴다. 가중치·공개 데이터는 데이터+코드에서 재생성 가능하므로 제외.

## 5) 실험 기록 확인 (MLflow)

```bash
~/rnd-env/bin/mlflow ui --backend-store-uri sqlite:///mlflow.db --port 5000
# → http://localhost:5000 (실험: 텍스트분류-범용, YOLO-커스텀)
```
