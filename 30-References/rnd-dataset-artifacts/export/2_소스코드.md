## 파일 구성

| 파일 | 역할 |
|---|---|
| `train_text.py` | **텍스트 분류 범용 학습기** — `text,label` CSV → 학습 → 산출물 3종 세트 |
| `classify_text.py` | **산출물 재사용 분류기** — 산출물 세트 로드 → 새 텍스트 분류 |
| `make_yolo_dataset.py` | **YOLO 데이터셋 규약 스캐폴더** — 규약 폴더 생성(+합성 데모 데이터) |
| `yolo_train_custom.py` | **YOLO 커스텀 범용 학습기** — `data.yaml` 규약 → 학습 → 산출물 3종 세트 |
| `make_pii_schema.py` | **PII 분류 체계 추출기** — ko-pii 33라벨 → `pii_label_schema.md/json` |
| `validate_pii_samples.py` | **PII 라벨 검증기** — 평가 데이터의 라벨이 33라벨 체계에 속하는지 강제 |
| `pii_label_schema.md/json` | 생성된 분류 체계 정본 (손으로 고치지 않음 — 재실행으로 갱신) |
| `requirements.txt` | 검증 당시 패키지 버전 |

## 핵심 설계 — 계약(Contract) 2개

**입력 계약** (데이터가 지켜야 할 모양):

```
텍스트: CSV 1개(text·label 열 자동감지, 70:10:20 자동분할)
        또는 디렉터리(train.csv/validation.csv/test.csv)
YOLO:   <데이터셋>/data.yaml + images/{train,val}/ + labels/{train,val}/
PII:    문장<TAB>라벨1,라벨2  (라벨은 33라벨 체계 강제)
```

**출력 계약** (산출물 3종 세트 — `artifacts/<이름>/`):

```
model.pt(또는 best.pt)  학습된 가중치
label_map.json          숫자↔클래스명 매핑 (없으면 가중치 재사용 불가)
meta.json               베이스모델·파라미터·지표·MLflow run id (재현 정보)
```

## 주요 코드

### train_text.py — 클래스 자동 인식 (확장성의 핵심)

```python
def find_columns(df):
    """열 이름에서 text/label 열을 자동 감지 (대소문자 무관)."""
    for c in df.columns:
        lc = str(c).lower()
        if "text" in lc and text_col is None:  text_col = c
        elif "label" in lc and label_col is None:  label_col = c

# 라벨 체계 자동 구축 — 문자열이든 숫자든 정렬해 고정 번호 부여
classes = sorted({str(v) for d in (train_df, val_df, test_df) for v in d[label_col]})
label2id = {c: i for i, c in enumerate(classes)}
model = AutoModelForSequenceClassification.from_pretrained(BASE, num_labels=len(classes))
```

새 데이터의 클래스가 2개든 7개든 자동으로 맞춰지므로, **스크립트 수정 없이 CSV만 바꾸면** 새 분류기가 나온다.
