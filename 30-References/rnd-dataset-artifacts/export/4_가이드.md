# 4. 가이드 (함정·재현 절차) — 학습 데이터 산출물 환경 (2026-07-08)

## 실제로 겪은 함정

### ★ MLflow 3.x — 파일 저장소(`./mlruns`)는 유지보수 모드
`mlflow.set_experiment()`가 파일 저장소로 해석되면
`The filesystem tracking backend ... is in maintenance mode` 예외가 난다.
특히 **ultralytics 학습 뒤에 MLflow를 쓰면** 프로세스 안에서 추적 URI가 바뀌어 있을 수 있다.
→ 두 학습기 모두 `mlflow.set_tracking_uri("sqlite:///<스크립트옆>/mlflow.db")`로 **명시 고정**했다.
실행 위치와 무관하게 항상 같은 DB에 기록된다.

### transformers 로드 리포트의 MISSING은 정상
`classifier.weight | MISSING` 경고는 **분류 헤드가 새로 초기화됐다**는 뜻 — 사전학습 모델에는
우리 과제의 분류층이 없으므로 당연하다. 그 헤드를 학습시키는 것이 파인튜닝이다.
(`UNEXPECTED`인 `cls.*`는 사전학습 전용 층이라 버려지는 것 — 역시 정상.)

### 숫자 라벨 데이터셋이면 분류 결과도 숫자로 나온다
`label_map.json`은 데이터에 있던 라벨을 그대로 보존한다. 기존 스팸 CSV처럼 라벨이 `0/1`이면
분류 출력도 `[0]/[1]`이다. **새로 만드는 데이터는 문자열 라벨(`스팸`,`정상`)을 권장** —
산출물만 봐도 의미가 읽힌다.

### YOLO 신규 클래스는 몇 에폭으로 안 배워진다
사전학습(yolov8n)에 없는 새 클래스(합성 도형)는 5에폭 시 mAP50 0.06 수준.
소량 데이터 기준 **40에폭**을 기본값으로 잡았다. 에폭당 수 초(소량 데이터·MPS)라 부담 없다.

### ultralytics `val()`은 project를 따로 줘야 한다
`train()`에 `project=`를 줘도 `val()`은 기본 `runs/detect/`에 저장한다 → 잔여 폴더 발생.
`val(project=..., name=...)`을 명시해 `yolo_runs/` 아래로 모았다.

### 기타
- `yolov8n.pt`·`klue/bert-base` 최초 실행 = 인터넷 필요 (이후 캐시).
- 장치는 자동 감지: Apple Silicon = MPS, 아니면 CPU.
- 스모크는 `LIMIT=300 EPOCHS=1`로 먼저 완주 — 작게 시작해 실패 비용을 줄인다.

## 재현 절차 (처음부터 끝까지)

```bash
cd 30-References/rnd-dataset-artifacts/export

# 0) 환경
python3.12 -m venv ~/rnd-env && ~/rnd-env/bin/pip install -r requirements.txt

# 1) PII 분류 체계 + 라벨 검증
~/rnd-env/bin/python make_pii_schema.py          # 33라벨/5그룹, 커버리지 10/33
~/rnd-env/bin/python validate_pii_samples.py     # ✅ 통과 (오타 라벨이면 종료코드 1)

# 2) 텍스트 분류 스모크 (기존 한국어 스팸 재사용)
DATA=../../rnd-bert-labeling-test/export/ko LIMIT=300 EPOCHS=1 NAME=ko-spam-smoke \
  ~/rnd-env/bin/python train_text.py             # 정확도 ~0.94
ARTIFACT=artifacts/ko-spam-smoke ~/rnd-env/bin/python classify_text.py "무료 쿠폰 당첨!"

# 3) YOLO 규약 end-to-end (라벨링 없이 합성 데모)
DEMO=1 ~/rnd-env/bin/python make_yolo_dataset.py
DATA=datasets/yolo-demo/data.yaml ~/rnd-env/bin/python yolo_train_custom.py
```
