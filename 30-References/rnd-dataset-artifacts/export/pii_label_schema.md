# PII 분류 체계 (라벨 스키마)

- 출처: **ko-pii 1.15.2** 내장 라벨 정의(`ko_pii.labels`) — 코드에서 자동 추출 (`make_pii_schema.py`)
- 전체 **33개 라벨**, 탐지 방식 **5개 그룹**
- 평가 데이터(`pii_samples.txt`) 커버리지: **10/33** (✅=샘플에 기대 라벨로 등장, ▫️=미커버 → 샘플 확장 후보)

## 결정적 검증 (8개)

| 커버 | 라벨 | 한국어명 | 검증 방식 |
|---|---|---|---|
| ✅ | `RRN` | 주민등록번호 | 13자리+날짜+체크섬(2020+ 는 신뢰도 신호) |
| ▫️ | `FRN` | 외국인등록번호 | 성별자리 5-8 + 체크섬 |
| ✅ | `BUSINESS_REG` | 사업자등록번호 | 국세청 가중합 체크섬 |
| ▫️ | `CORP_REG` | 법인등록번호 | 법인 체크섬 (RRN 우선) |
| ▫️ | `DRIVER_LICENSE` | 운전면허번호 | 지방청 코드 화이트리스트 |
| ✅ | `PASSPORT` | 여권번호 | prefix(M/S/PP 등) + 8자리 |
| ✅ | `CARD` | 카드번호 | BIN 화이트리스트 + Luhn |
| ▫️ | `PNU` | 토지번호 | 19자리 + 시·도 코드 |

| ▫️ | `MEDICAL_INSURANCE` | 건강보험증번호 | 건강보험/보험증 키워드 |

| ▫️ | `EMPLOYEE_ID` | 사번 | 사번/직원번호/임용번호 키워드 |

## 사용 규칙

- 평가 데이터(`pii_samples.txt`)의 기대 라벨은 **반드시 위 33개 중 하나** — `validate_pii_samples.py`가 강제한다.
- 새 샘플을 추가할 때는 ▫️(미커버) 라벨을 우선 채워 커버리지를 넓힌다.
- 이 문서는 손으로 고치지 않는다 — ko-pii 업그레이드 시 `make_pii_schema.py` 재실행으로 갱신.
