# 00 학습메모 — 통합 패키지 재구축·발송 (2026-07-19~20)

> 진행 기록: [[로드맵]] · 대상: 발송본 불명 사태 → rnd-detection-stack 재구축 → 리허설 → 발송까지의 교훈.

## 1. 스냅샷은 원본을 따라오지 않는다 — 복사본과 링크의 구분

- 발송 폴더는 build.py가 조립 순간에 뜬 **복사본** — 원본(모듈 export·통합 문서)을 고쳐도 자동 반영 안 됨.
  심볼릭 링크(팻말)와 정반대. 원본 수정 → **재조립 → zip 재생성 → 배포 위치 교체**가 한 세트.
- 최신 여부 판별법: 임시 폴더에 새로 조립해 기존 스냅샷과 `diff -r` — 차이 0이면 최신. 발송 직전 관문으로 채택.

## 2. 도구의 기본값이 시간에 의존하면 함정이 된다

- build.py 기본 출력이 `…보고-<오늘날짜>` — 날짜가 바뀌자 **새 폴더**를 만들고, 옛 폴더를 압축해
  구판 zip을 배포할 뻔함. 같은 보고를 여러 날 다듬을 땐 `OUT=` 으로 기존 스냅샷 강제 지정.
- 일반화: 재실행이 같은 결과를 내야 하는 도구에서 `today()` 류 기본값은 멱등성을 깨는 지점.

## 3. zip은 표준이 하나가 아니다 — 한글 파일명 크로스플랫폼 실측

| 생성 도구 | 윈도우 내장 해제기 | 맥 | 원인 |
|---|---|---|---|
| 리눅스 `zip -r` (Info-ZIP) | ❌ "올바르지 않음" (Expand-Archive 0개) | ✅ | 윈도우 해제기 호환성 |
| PowerShell Compress-Archive | ✅ | ⚠️ 위험 | 경로 구분자 **백슬래시** |
| 윈도우 tar(bsdtar) | ✅ | ❌ 깨짐 | 파일명 **CP949**, EFS 플래그 없음 |
| **python zipfile** | ✅ | ✅ | UTF-8 + EFS 플래그 + 슬래시 |

- 결론: 발송 zip은 `python -m zipfile -c`로만. 검증법: `zipfile.ZipInfo.flag_bits & 0x800`(EFS)·백슬래시 유무.

## 4. 경로 하드코딩은 "내 환경 증명"일 뿐 — 이식성의 3층 방어

- `~/rnd-env/bin/python` 하드코딩 → 받는 사람이 README대로 패키지 안에 venv를 만들면 실패,
  윈도우는 `Scripts\python.exe` 구조라 무조건 실패. 처방: **자기 자신은 `sys.executable`**(어디서 실행하든 맞음),
  타 환경은 **OS 분기(bin/Scripts) + 환경변수 오버라이드(`OCR_ENV=`)**.
- 문서 속 실행 예(`~/rnd-env/bin/python xxx.py`)도 같은 문제 — 코드가 안 죽어도 **안내 불일치**가 사람을 막는다.

## 5. 문서의 버전 주장은 실측으로만 — "최소 3.10"의 거짓

- requirements 머리말의 "최소 3.10"은 numpy 2.5.1(≥3.12 요구) 때문에 실제로 성립 안 함 — 3.10 설치가
  즉시 실패. 고정 버전 조합의 지원 범위는 **가장 좁은 패키지가 정한다**. 표기는 "3.12 필수"로,
  문제 해결 표엔 실패 증상(`No matching distribution…`)을 그대로 적어 검색이 닿게.

## 6. NFD 한글 폴더명 — 맥↔윈도우 왕복의 조용한 유실

- 맥에서 커밋된 폴더명이 NFD(자모 분해형)로 저장 → WSL 체크아웃 후 작업 트리에서 파일들이
  **git status상 D(삭제)로 유실**되는 현상 2회 관찰(원인 도구 미특정). `git restore`로 복구 가능하나,
  새 산출물 폴더명은 ASCII 우선 검토 + 발송은 zip으로(폴더 직접 전달 금지).
- 쉘에서도 NFC로 친 글롭이 NFD 이름과 안 맞음 — ASCII 부분(`*R?D*`)으로 매칭하는 우회.

## 7. 같은 wheel, 다른 CPU — paddle oneDNN 커널 버그

- paddlepaddle 3.3.1이 이 CPU(WSL)에서 `NotImplementedError(ConvertPirAttribute2RuntimeAttribute)` —
  맥에선 멀쩡. `enable_mkldnn=False`로 우회(채점은 CPU 고정이라 속도 무관). FLAGS 환경변수는 안 먹혔고
  **생성자 옵션**이 정답이었음 — 우회는 라이브러리가 공식으로 여는 문부터.

## 8. WSL은 윈도우가 아니다 — 경계에서 생긴 일들

- zsh에서 `py`·`Activate.ps1` 없음(윈도우 전용) — README_windows는 **PowerShell 기준**임을 문서에 명시할 것.
- 바탕화면이 OneDrive로 리디렉션(`C:\Users\<u>\OneDrive\Desktop`) — `~/Desktop`(WSL)과 3중으로 다른 장소.
  대용량 venv를 OneDrive 안에 만들면 동기화 폭탄 — 리허설은 `C:\work` 등 로컬에.
- WSL에 한글 폰트 없음 → 렌더된 도식 한글이 전부 □ (조용한 실패). `/mnt/c/Windows/Fonts/malgun.ttf`를
  `~/.fonts`에 복사 + `fc-cache`로 해결. PowerShell 첫 .ps1은 ExecutionPolicy(RemoteSigned)가 관문.

## 9. 리허설의 정의 — "내가 만든 환경에서 돌아감" ≠ "문서만으로 돌아감"

- 검증 2단계: ① 내 환경에서 기능 검증 → ② **복사본**에서 README 절차만으로 처음부터(새 venv,
  가중치 재생성 포함). ②에서만 잡히는 결함이 있었음(venv 위치, csv 동봉, 버전 표기).
- 리허설은 반드시 복사본에서 — 발송 스냅샷 폴더 안에 venv·생성물을 만들면 기록이 오염된다.

## 10. 화면 문구는 독자에 맞춰 두 겹으로 — 그리고 침묵은 버그다

- R&D 용어("교차 검증·합의·검수")를 초심자용 일상어("둘의 답안 맞춰보기·다르게 읽은 단어만 사람이 확인")로
  풀고, 전문 용어는 괄호 병기 — 문서와의 연결은 유지.
- 첫 클릭 수십 초 무반응(모델 지연 로딩)은 사용자에겐 고장 — 클릭 즉시 ⏳ 표시 후 결과로 교체.
  단, gradio 이미지·갤러리 출력은 자체 로딩 표시가 있어 **Markdown 전용 출력에만** 필요했음 (전부 바르지 말 것).

## 11. 발송본 불명의 구조적 해결 — 이름·스냅샷·조립기

- 재발 방지 3장치: ① 산출물 폴더에 **날짜 명명** ② 발송 직전 diff 검증 ③ 발송 후 스냅샷을
  **99-Archive로 이동**(= "상사가 받은 것"의 정확한 기록). 정본은 조립기의 집(rnd-detection-stack) 한 곳,
  날짜 폴더는 실행 결과 — 도구는 한 곳에서 진화, 산출물은 날짜별 동결.
