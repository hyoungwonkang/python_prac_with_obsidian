# 4. 가이드 — 실무 판단 기준

새 도메인·새 과제를 만났을 때 참고하는 판단 원칙 총정리 (실측 근거).

## 1. 분류 방법 선택 — 어느 실수가 치명적인가
실수는 둘뿐: **오탐**(정상을 건드림, 정밀도↓) / **미탐**(대상을 놓침, 재현율↓).
| 업무 성격 | 권장 | 근거 |
|---|---|---|
| 판정 뒤 차단·제재 자동 (억울한 피해 불가) | HYBRID-AND | 정밀도 1.0, 오탐 0 |
| 놓치면 사고 (회수가 생명) | HYBRID-OR | 재현율 최고 |
| 균형·사람 검수 뒤따름 | BERT 단독 | F1 최고 |
| 무서버·실시간·근거 제시 의무 | RULE 단독 | 0.12초, 걸린 키워드 제시 |

## 2. 항상: 테스트 누출 금지 3단
**만들 때 train → 조정 validation → 채점만 test.** 룰 키워드·문턱·프롬프트도 모두 이 규율.
test를 보고 정하면 그 순간 비교가 무효. (오답 정성 분석도 test를 본 것 → 개선 아이디어는 train/val에서 재확인 후 반영.)

## 3. 앙상블은 오답 교집합부터
**AND의 오탐 = 오탐 교집합 / OR의 미탐 = 미탐 교집합.** 두 방법의 오답이 안 겹칠수록 결합 이득이 크다. 거의 겹치면 결합 대신 더 나은 단일 방법을 찾는다.

## 4. 닫힌 집합과 캐치올(기타) 클래스
분류기는 후보에 없는 입력도 **가장 덜 어색한 후보로 억지 배정**한다(거부 못 함). 대책:
- "일반 대화"·"일상 사진" 같은 **기타 후보**를 반드시 포함 (억지 배정의 무해한 착지점)
- CLIP 프롬프트: 부정형 금지("no people"은 동전 이하), **대비쌍**(찾는 것 vs 그 외 긍정 묘사), 흔한 장면은 **전용 후보**(포괄 후보는 특정성 싸움에서 짐), 장면을 **완전히** 서술한 문구가 이긴다
- **후보 목록 = 탐지 범위의 정본** — 데이터셋처럼 버전 관리. 새 위험 유형은 모델 재학습이 아니라 문장 추가로 대응.

## 5. PII — 민감도(항목)와 종합 위험도(문장)는 다르다
- **민감도** = 유형의 등급(오탐이어도 붙음) / **종합 위험도** = 이 문장의 개인정보만으로 개인이 특정되는가
- 식별자(주민번호 등, 1개로 즉시 식별)는 종합 최고 직행 / 준식별자(이름·전화 등)는 결합될수록 상승
- 룰 기반은 형식(체크섬) 강, 인명 약 → NER과 하이브리드 (인명이 두 모듈의 접점)

## 6. 파인튜닝 방식 — 지식을 어디에 넣나
| 방식 | 지식 위치 | 새 개념 추가 |
|---|---|---|
| CLIP zero-shot | 프롬프트(입력) | 문장 한 줄 (재학습 0) |
| linear probe | 얇은 머리 | 소량 라벨 학습 |
| YOLO/BERT 파인튜닝 | 가중치 | 라벨링+재학습 |
- **소데이터 병합 재학습 주의**: 신규 클래스 몇 장으로 통합 재학습하면 **파국적 망각**(기존 대량 지식 붕괴). 실무 정답은 범용+특화 **모델 병행 운용**(데모의 YOLO_PT 교체 방식).
- CLIP은 "재학습 없이 체계 교체"의 장점과 "말 안 한 건 못 찾음"의 한계가 동전의 양면.

## 7. OCR — ②추출 품질이 ③탐지 성능이다
하류(분류·NER·PII)는 OCR이 준 텍스트만 본다(이미지 속 글자는 텍스트 모델에게 안 보임).
- **형식 과제(전화번호 등 PII)는 한 글자 오독에 전멸**하고 실패는 조용하다(위험도 강등 — 검수 대상에서도 빠짐). 중복 단서 과제(스팸·NER)는 견딘다 → 무너지기 쉬운 하류부터 상류 품질 보강.
- 엔진 선택: 정확도(형식 민감·증거)=Paddle / 처리량=Easy(5~10배 빠름) / 고신뢰=두 엔진 교차 검증(오답 프로필 상이 → 불일치만 검수, 맹점 0 실측).
- **전처리는 표적 치료**: 기울기 보정은 상시 무해+극적(엔진 무관) / 확대 등은 유령 글자 위험 — before·after 채점 후만. 병 없는 이미지엔 무익.
- 방어선 3층: 전처리(열화 제거) → 신뢰도 게이트(자기 신고 — 낮으면 검수 큐) → 앙상블 불일치(타 엔진 증언 — 신뢰도가 못 잡는 확신형 오탐 포획). 검수 문턱은 validation으로.

## 8. 데이터 순환 (검수→편입)
모델 판정은 **초안(의사 라벨)**, 사람 확정으로만 정답 승격. 검수 예산은 판정이 갈린 경계 사례에 집중. 확정분은 `text,label` 계약으로 학습기에 바로 편입 → 순환 폐쇄. **소량 검수분은 단독 학습 말고 기존 train에 편입**(고정 val·test는 불변 — 잣대 유지).
