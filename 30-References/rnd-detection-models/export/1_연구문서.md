# 연구문서 — NER · YOLO · PII 탐지 모델 R&D

작성일: 2026-07-05 · 검증 환경: macOS / Apple Silicon (M4 Max, PyTorch 2.12 MPS) · Python 3.12

## 0. 요약

탐지 플랫폼의 **1차 탐지 계층** 3종을 공개 데이터·로컬 환경에서 코드로 재현했다.

| 모델 | 계층 대응 | 방식 | 데이터 | 핵심 결과 |
|---|---|---|---|---|
| **NER** | NER Agent (인명·기관·지명 등 추출) | `BertForTokenClassification` 파인튜닝 | KLUE-NER | 엔티티 F1 **0.7057** |
| **YOLO** | Image Analyzer (이미지 객체 탐지) | YOLOv8n 전이학습 | COCO128 | mAP50 **0.606** / mAP50-95 **0.450** |
| **PII** | 개인정보 비식별화 | ko-pii 룰+체크섬 라이브러리 | (학습 없음) | 커버리지 **90.9%** |

세 모델 모두 별도 GPU 없이 로컬(MPS)에서 미니 데이터로 end-to-end 실행·검증했다.
NER·YOLO는 사전학습 가중치에서 **전이학습**(무거운 본체 재사용 + 소량 미세조정), PII는
학습이 아니라 **규칙 기반 라이브러리 통합**이라는 성격 차이가 있다.

## 1. NER (개체명 인식)

### 개요
NER은 문장에서 인명·기관·지명 등 **개체(entity)에 태그를 다는** 토큰 단위 분류다.
BERT 본체 위에 **토큰마다** 분류층을 얹은 `BertForTokenClassification` = "BERT 본체 + NER용 분류층".
(문장 하나에 라벨 하나인 텍스트 분류와 달리, **토큰 개수만큼** 라벨을 낸다.)

### 데이터·라벨링 (KLUE-NER)
- **KLUE-NER**: 한국어 개체명 공개 데이터(HuggingFace `datasets`, `klue/klue` config `ner`).
- 태그 **13종** = 6개 유형 × BIO + O: PS(인명)·LC(지명)·OG(기관)·DT(날짜)·TI(시간)·QT(수량).
- **subword 정렬 + -100**: BERT는 단어를 subword로 쪼개므로, 각 문자(단어)의 **첫 subword에만
  라벨을 주고 나머지 subword·특수토큰·패딩은 -100**으로 채워 손실에서 제외한다. → 분류 파인튜닝에서
  배운 ignore_index(-100)가 NER의 subword 정렬에 그대로 재등장.

### 검증 결과 (KLUE-NER validation 1,200건, 학습 미사용)
학습 6,000건 · 3 epoch · klue/bert-base · 로컬 MPS.
```
entity F1: 0.7057   (micro precision 0.6860 / recall 0.7266)

유형별 F1:  QT 0.828  TI 0.855  DT 0.766  PS 0.725  OG 0.561  LC 0.499
```
- 수량(QT)·시간(TI)처럼 형식이 뚜렷한 유형이 높고, 지명(LC)·기관(OG)처럼 문맥 의존이 큰 유형이 낮다.
- 추론 예시(`홍길동은 2024년 3월에 서울시청 앞에서 김철수를 만났다.`):
  `홍길동`[PS] · `2024년3월`[DT] · `서울시청`[LC] · `김철수`[PS] 정상 추출.

## 2. YOLO (객체 탐지)

### 개요
YOLO는 이미지에서 **객체의 위치(박스) + 종류**를 한 번에 예측하는 실시간 객체 탐지 모델이다.
`ultralytics` 패키지로 사전학습 가중치(YOLOv8n)를 불러와 **전이학습**한다 —
BERT 파인튜닝과 같은 발상(무거운 본체 재사용, 소량 데이터로 미세조정).

### 데이터 (COCO128)
- **COCO128**: COCO에서 128장을 추린 공식 스모크 데이터. 가중치(`yolov8n.pt`)·데이터 모두
  첫 실행 시 자동 다운로드.
- YOLOv8n = 가장 작은 변형(약 320만 파라미터, 저장 6.5MB) — 로컬 실습에 적합.

### 검증 결과 (COCO128, 1 epoch 전이학습, MPS)
```
mAP50-95: 0.4501
mAP50   : 0.6063
```
- mAP(mean Average Precision) = 객체 탐지의 표준 지표. mAP50은 IoU 0.5 기준, mAP50-95는 더 엄격한 평균.
- yolov8n이 이미 COCO로 사전학습돼 있어 COCO128에서는 1 epoch만으로도 높은 값. 실제 과제(위험 이미지)
  에서는 도메인 라벨로 추가 전이학습 필요.
- 추론 예시(샘플 이미지): `bowl`(0.92)·`broccoli`(0.80)·`hot dog`(0.63) 등 6개 객체 탐지, 주석 이미지 저장.

## 3. PII (개인정보 탐지·비식별화)

### 개요
PII(개인식별정보) 탐지는 문장에서 주민번호·전화·계좌·인명 등을 찾아 **마스킹(비식별화)** 하는 작업.
여기서는 학습 모델이 아니라 **`ko-pii`** 라이브러리(룰 + 사전 + 체크섬, MIT)를 통합했다.
플랫폼의 NER Agent가 추출하는 인명·연락처·계정이 곧 PII이며, 룰 엔진 계층에 해당한다.

### 방식 (ko-pii)
- **규칙·형식·체크섬 3중**: 주민번호·사업자번호·카드는 **체크섬 검증**으로 오탐을 줄이고,
  전화·이메일·IP·계좌는 형식 검증, 인명·주소는 사전/휴리스틱.
- **33개 PII 범주** 지원. STRICT 모드 + tokenize 전략으로 `<PERSON_1>`·`<RRN_1>` 등 되돌리기 가능한
  토큰으로 치환하고, 원본은 Vault에 보관(`reveal`로 복원). 위험도(RiskLevel)도 함께 산출.

### 검증 결과 (손수 라벨 샘플 11개 유형, 8문장)
```
커버리지: 10/11 = 90.9%  (기대 PII 유형 중 탐지된 비율)
```
- 정상 탐지: PERSON·RRN(체크섬)·PHONE·EMAIL·ACCOUNT·BUSINESS_REG·VEHICLE·PASSPORT·CARD·IP·ADDRESS.
- **미탐 1건**: `김영희`(PERSON) — 문맥에 따라 사전 기반 인명 탐지가 놓칠 수 있음(룰 방식의 구조적 한계).
  → 실전에서는 ko-pii(룰) + NER(학습 모델)을 **하이브리드**로 결합해 상호 보완하는 것이 바람직.

---

## 4. 결론

- 플랫폼 1차 탐지 3계층(NER·이미지·PII)을 **로컬·미니데이터·공개 데이터**로 end-to-end 재현 완료.
- **전이학습(NER·YOLO) vs 규칙 라이브러리(PII)** 두 접근을 한 번에 비교 — 학습형은 도메인 데이터로 성능
  향상 여지가 크고, 규칙형은 즉시 사용 가능하나 문맥 의존 개체에 약하다. **하이브리드**가 실전 방향.
- 다음 단계: 도메인 라벨 데이터 확보 → 정식 학습(대용량/다 epoch), NER+PII 하이브리드 파이프라인.
