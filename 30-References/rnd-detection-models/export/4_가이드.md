# 가이드 — 재현 절차 & 주의점

## A. 처음부터 재현 (체크리스트)
- [ ] 1. `brew install python@3.12` → `~/rnd-env` venv 생성 → 패키지 설치(§사용법 0)
- [ ] 2. MPS 확인: `python -c "import torch;print(torch.backends.mps.is_available())"` → True
- [ ] 3. **NER**: `NER_SUBSET=6000 EPOCHS=3 python finetune_ner.py` → F1 ~0.70 → `predict_ner.py`
- [ ] 4. **YOLO**: `EPOCHS=1 python yolo_train.py` → mAP50 ~0.6 → `yolo_predict.py`
- [ ] 5. **PII**: `python pii_detect.py` → 마스킹 확인 → `pii_eval.py` → 커버리지 ~91%
- [ ] (선택) MLflow 서버 띄우고 `eval_mlflow_ner.py`로 지표 기록

## B. 모델별 재현 지점
| 모델 | 재현 위치 |
|---|---|
| NER | `finetune_ner.py` 학습 로그의 `entity F1` + 유형별 seqeval 표 |
| YOLO | `yolo_train.py` 말미 `mAP50 / mAP50-95` + `yolo_runs/` 주석 이미지 |
| PII | `pii_eval.py` 커버리지 + `pii_detect.py` 마스킹 출력 |

## C. 주의점 (함정)
1. **Python 버전** — ko-pii는 **3.10+ 필수**, ultralytics도 3.9는 위험. 반드시 3.12 venv에서.
   기존 3.9.6 `~/ml-env`와 혼동 금지.
2. **datasets 5.x 변경** — 스크립트형 `load_dataset("klue", ...)`는 폐기. `"klue/klue"` 데이터 repo + config `"ner"` 사용.
3. **NER subword 정렬** — 첫 subword만 라벨, 나머지 `-100`. 정렬이 어긋나면 F1이 왜곡되므로,
   의심되면 소량으로 `word_ids()` 매핑을 눈으로 확인.
4. **KLUE 공식 test셋은 라벨 비공개** — validation을 평가셋으로 사용(리더보드 test는 라벨 없음).
5. **YOLO mAP가 높은 이유** — yolov8n이 이미 COCO 사전학습됨 → COCO128 1 epoch도 높게 나옴.
   실제 과제(위험 이미지)는 **도메인 라벨로 추가 전이학습** 필요(이 수치를 실전 성능으로 오해 금지).
6. **PII 룰의 한계** — 주민번호·카드는 체크섬으로 강하지만, **인명·주소 등 문맥 의존 개체는 놓칠 수 있음**
   (실측: `김영희` 미탐). 룰(ko-pii) + 학습모델(NER) **하이브리드**로 보완 권장.
7. **MPS 비결정성** — 시드 고정해도 소수점은 약간 흔들릴 수 있음. 또 과거 MPS 비연속 가중치 버그
   사례가 있으니 결과가 이상하면 CPU와 소량 비교.
8. **산출물 위치** — ultralytics가 repo 루트에 `datasets/`·`yolo_runs/` 생성, `.pt`는 대용량
   (ner_klue.pt ~440MB). git 커밋 시 gitignore 필수.

## D. 확장 방향
| 목표 | 방법 |
|---|---|
| NER 성능↑ | 전체 KLUE-NER(약 21k) + 다 epoch, 또는 도메인(성착취 정황) 라벨 데이터 |
| PII 정확도↑ | ko-pii(룰) + NER(학습) 하이브리드 — 상호 오탐/미탐 보완 |
| YOLO 실전화 | Roboflow 등 도메인 라벨셋으로 커스텀 클래스 전이학습 |
| 이미지 전처리 연계 | OpenCV(프레임 추출·얼굴 비식별화) → YOLO 파이프라인 |
| 실험 추적 | 모든 학습 MLflow 기록(한글 키 관례) → run 비교 |

## E. 검증 환경
- macOS / Apple Silicon (M4 Max) / Python **3.12.13** (`~/rnd-env`).
- `torch 2.12.1`(MPS) · `transformers 5.13.0` · `datasets 5.0.0` · `seqeval` · `ultralytics 8.4.87` · `ko-pii 1.15.2` · `mlflow`(선택, 실험 기록).
- TensorFlow 불필요. 별도 GPU 없이 로컬에서 3모델 미니데이터 end-to-end 완료(각 수 분~십수 분).
- 무거운 정식 학습이 필요하면 Colab GPU로 이전 가능하나 **자동 전환하지 않고 먼저 협의**한다.
