## 1. 통합 아키텍처도 — 세 모듈이 탐지 파이프라인에서 서는 자리

```mermaid
flowchart TB
    IN[수집 데이터] --> TXT[텍스트]
    IN --> IMG[이미지]

    subgraph DETECT[1차 탐지 계층]
        subgraph TEXTMOD[텍스트 분석]
            PII["PII 모듈 (ko-pii)<br/>룰·사전·체크섬 — 학습 없음"]
            NER["NER 모듈 (klue/bert-base)<br/>토큰 분류 — 파인튜닝"]
        end
        subgraph IMGMOD[이미지 분석]
            YOLO["YOLO 모듈 (YOLOv8n)<br/>객체 탐지 — 전이학습"]
        end
    end

    TXT --> PII
    TXT --> NER
    IMG --> YOLO

    PII --> M["마스킹 문장 + Vault(복원 금고)<br/>위험도(CRITICAL 등)"]
    NER --> E["개체 목록<br/>인명·기관·지명·날짜·시간·수량"]
    YOLO --> B["객체 박스 목록<br/>위치 + 클래스 + 신뢰도"]

    NER -.->|"인명 보완 (하이브리드)"| PII

    M --> OUT[결과 통합 → 위험 판정·검수 계층으로]
    E --> OUT
    B --> OUT
```

```
[텍스트 없이 보는 판]

수집 데이터
 ├─ 텍스트 ─┬─ [PII: ko-pii 룰+사전+체크섬] ──→ 마스킹 문장 + Vault + 위험도
 │          └─ [NER: klue/bert-base 토큰분류] ─→ 개체 목록 (인명·기관·지명…)
 │                └······ 인명 보완 (하이브리드: 룰의 인명 미탐을 NER이 메움)
 └─ 이미지 ─── [YOLO: YOLOv8n 객체탐지] ─────→ 박스 목록 (위치+클래스+신뢰도)
                                  ↓
                    결과 통합 → 위험 판정·검수 (후속 계층)
```

- **분업**: 텍스트는 PII(형식 있는 개인정보)와 NER(문맥 의존 개체)이 나눠 맡고, 이미지는 YOLO가 맡는다.
- **하이브리드 근거(실측)**: PII 룰은 주민번호·전화(체크섬·형식)에 강하나 인명에서 미탐 발생(김영희) → 인명은 NER이 보완.

## 2. 모델별 아키텍처도 (내부 구조)

### 2-1. NER — BERT 본체 + 토큰 분류층

```mermaid
flowchart LR
    S["문장<br/>홍길동은 서울시청에서…"] --> T["토크나이저<br/>(글자 단위, WordPiece)"]
    T --> BB["BERT 본체 klue/bert-base<br/>어텐션 12층 × 12헤드 (768차원)<br/>— 사전학습 재사용 —"]
    BB --> H["토큰 분류층 (신규)<br/>768 → 13"]
    H --> TAG["글자마다 BIO 태그<br/>B-PS I-PS I-PS O B-LC …"]
    TAG --> ENT["구간 병합 → 개체<br/>홍길동[인명] 서울시청[지명]"]
```

- 13 = 6개 유형(인명 PS·지명 LC·기관 OG·날짜 DT·시간 TI·수량 QT) × 시작 B/이어짐 I + 개체아님 O.
- 전이학습: 본체(1.1억 파라미터)는 재사용, 분류층만 새로 얹어 소량 데이터로 미세조정.

### 2-2. YOLO — 사전학습 탐지기 전이

```mermaid
flowchart LR
    I["이미지 640×480"] --> Y["YOLOv8n<br/>(백본→넥→헤드, 약 320만 파라미터)<br/>— COCO 사전학습 재사용 —"]
    Y --> C["후보 박스 8,400개<br/>각각 좌표4 + 80클래스 점수"]
    C --> F["신뢰도 필터(≥0.25)<br/>+ NMS(중복 제거)"]
    F --> O["최종 박스<br/>bowl(0.92) broccoli(0.80) …"]
```

- 박스 = 중심x·중심y·너비·높이(이미지 대비 비율) + 클래스 + 신뢰도.
- 신뢰도 = 80클래스 점수 중 최고값(로짓→시그모이드).

### 2-3. PII — 규칙 엔진 (학습 없음)

```mermaid
flowchart LR
    S["문장<br/>홍길동 880101-1234568 …"] --> A

    subgraph A["Anonymizer (STRICT + tokenize)"]
        direction TB
        D["탐지부<br/>① 룰(정규식): 전화·이메일·계좌<br/>② 사전+휴리스틱: 인명·주소<br/>③ 체크섬: 주민번호·카드·사업자"]
        P["처리부: 유형별 번호표 치환<br/>&lt;PERSON_1&gt; &lt;RRN_1&gt; …"]
        V["보관부: Vault<br/>원본 저장 → 권한 시 reveal 복원"]
        D --> P --> V
    end

    A --> O["마스킹 문장 + 탐지목록 + 위험도"]
```

- 33개 PII 범주. 체크섬 = 수학 검증(예: 주민번호 앞 12자리 가중합 mod 11 → 끝자리 일치)으로 오탐 억제.

## 3. 프로세스 처리 흐름 단계 도식도 (R&D 수행 절차)

### 3-1. NER (학습형)

```mermaid
flowchart LR
    A["① 데이터 준비<br/>KLUE-NER 자동 다운로드<br/>(글자 단위 BIO 라벨)"] --> B["② 라벨 정렬<br/>토큰화 + -100 처리<br/>(특수토큰·패딩 채점 제외)"]
    B --> C["③ 파인튜닝<br/>6,000건 × 3 epoch<br/>loss→역전파→가중치 조정"]
    C --> D["④ 평가<br/>검증 1,200건<br/>엔티티 F1 0.7057"]
    D --> E["⑤ 저장·추론<br/>ner_klue.pt (432MB)<br/>문장→개체 추출 확인"]
```

### 3-2. YOLO (전이학습형)

```mermaid
flowchart LR
    A["① 준비<br/>yolov8n.pt + COCO128<br/>자동 다운로드"] --> B["② 전이학습<br/>128장 × 1 epoch<br/>(MPS)"]
    B --> C["③ 평가<br/>mAP50 0.606<br/>mAP50-95 0.450"]
    C --> D["④ 저장·추론<br/>yolo_best.pt<br/>탐지 + 주석 이미지 확인"]
```

### 3-3. PII (통합형 — 학습 단계 없음)

```mermaid
flowchart LR
    A["① 설치<br/>pip install ko-pii<br/>(Python 3.10+)"] --> B["② 설정<br/>Anonymizer<br/>STRICT + tokenize"]
    B --> C["③ 탐지·마스킹<br/>치환 + Vault 복원<br/>동작 확인"]
    C --> D["④ 커버리지 평가<br/>11개 유형 중 10개<br/>= 90.9%"]
```

### 세 흐름 비교 (한눈에)

| 단계 | NER | YOLO | PII |
|---|---|---|---|
| 준비 | 데이터 자동DL + 라벨 정렬 | 가중치·데이터 자동DL | 설치만 |
| **학습** | ✅ 파인튜닝 (3 ep) | ✅ 전이학습 (1 ep) | **없음** (룰 기반) |

| 산출물 | ner_klue.pt | yolo_best.pt | (가중치 없음) |
| 실전 확장 | 도메인 라벨 재학습 | 커스텀 데이터 전이 | NER 하이브리드 |
