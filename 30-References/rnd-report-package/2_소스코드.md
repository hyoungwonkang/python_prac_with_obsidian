# 2. 소스코드 — 폴더별 코드 지도

실행 환경 공통(`rnd-env`, Python 3.12). 각 코드는 환경변수 계약으로 동작(도메인 교체 지점).

## rnd-dataset-artifacts/export
| 파일 | 역할 |
|---|---|
| `train_text.py` | 텍스트 분류 범용 학습기 — `DATA=CSV NAME=..` → 3종 세트 생성 |
| `classify_text.py` | 산출물 3종 세트만으로 재사용 분류 |
| `make_yolo_dataset.py` | YOLO 데이터셋 뼈대 생성 (`CLASSES=..`) |
| `webcam_capture.py` | 웹캠 촬영 도구 (라벨링용 사진 수집) |
| `labelme_to_yolo.py` | labelme JSON → YOLO 규약 변환 (라벨 체계 강제·오타 차단) |
| `yolo_train_custom.py` | 커스텀 YOLO 전이학습 → 3종 세트 |
| `make_pii_schema.py`·`validate_pii_samples.py` | PII 33라벨 스키마 추출·검증 |

## rnd-rule-vs-bert/export
| 파일 | 역할 |
|---|---|
| `rule_spam.py` | 룰 분류기 (키워드11+패턴4, train 통계로 선정·문턱 조정 내장) |
| `eval_compare.py` | 고정 test셋 동일 잣대 채점 (RULE + BERT 산출물, 학습 코드 불의존) |
| `analyze_errors.py` | 오답 교집합 정성 분석 + HYBRID-OR/AND |

## rnd-clip/export
| 파일 | 역할 |

## rnd-detection-models/export (기존 — 데모가 재사용)
| 파일 | 역할 |
|---|---|
| `predict_ner.py`·`ner_dataset.py` | NER 추론 (토큰 분류 + 구간 병합) |

| `pii_detect.py` | ko-pii 마스킹 데모 |

## 재사용 관계 (데모의 의존)

```
rnd-uxui-demo/app.py
 ├─ import rule_spam            ← rnd-rule-vs-bert/export
 ├─ import predict_ner          ← rnd-detection-models/export (+ ner_klue.pt)
 ├─ load ko-spam-full 3종 세트  ← rnd-dataset-artifacts/export/artifacts
 ├─ ko-pii (라이브러리)
 └─ KoCLIP (자동 다운로드)
```
데모는 상대경로(`HERE.parent.parent`)로 형제 폴더를 찾으므로 **이 폴더 구조를 유지**해야 한다.

## 공통 설계 원칙
- **3종 세트 규약**: 가중치 + label_map.json + meta.json — 코드 없이 재사용·비교 가능.
- **환경변수 계약**: DATA·NAME·PROMPTS·YOLO_PT 등으로 도메인·모델 교체 (코드 수정 0).
- **MLflow 2층**: 과정은 로컬 sqlite, 최종 지표는 통합 서버(`MLFLOW_URI=`).
- **장치 자동 감지**: MPS→CUDA→CPU.
