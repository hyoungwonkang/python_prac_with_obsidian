# 3. 사용법 — 실행법 (맥·윈도우 공통)

> 설치는 `README_mac.md` / `README_windows.md`(OS별) 참조. 아래 명령의 `python`은 활성화된 `rnd-env` 기준.
> Windows는 환경변수를 `$env:NAME="값";` 형식으로 (예: `$env:DATA="..."; python train_text.py`).

## 1) 텍스트 분류 — 새 분류기 만들기
```bash
cd rnd-dataset-artifacts/export
DATA=/경로/data.csv NAME=my-clf python train_text.py     # text,label CSV → 3종 세트
ARTIFACT=artifacts/my-clf python classify_text.py "분류할 문장"
```

## 2) 분류 방법 비교
```bash
cd rnd-rule-vs-bert/export
python rule_spam.py          # 룰 문턱 근거 (validation)
python eval_compare.py       # RULE + BERT 동일 잣대 채점
python analyze_errors.py     # 오답 교집합 + 하이브리드
```

## 3) CLIP 상황 판단
```bash
cd rnd-clip/export
BASE=Bingsu/clip-vit-base-patch32-ko \
PROMPTS="온라인 도박 사이트 화면|일상 생활 사진|음식 사진|문서나 서류" \
  python clip_zeroshot.py
```
프롬프트 수칙: 부정형 금지 · 대비쌍 · 캐치올(기타) 후보 포함 · 한국어는 KoCLIP.

## 4) 직접 라벨링 → YOLO 학습
```bash
cd rnd-dataset-artifacts/export
NAME=my-ds CLASSES=대상1,대상2 python make_yolo_dataset.py    # ① 뼈대
OUT=~/photos python webcam_capture.py                        # ② 촬영(웹캠) — 또는 직접 준비
labelme ~/photos                                            # ③ 라벨링(로컬 GUI)
JSONS=~/photos DATASET=datasets-own/my-ds SPLIT=train python labelme_to_yolo.py  # ④ 변환
DATA=datasets-own/my-ds/data.yaml EPOCHS=40 python yolo_train_custom.py          # ⑤ 학습
```

## 5) 통합 데모 (진입점)
```bash
cd rnd-uxui-demo/export
python app.py                                    # http://127.0.0.1:7860
YOLO_PT=<직접학습 best.pt> python app.py          # 커스텀 YOLO로 교체
```
탭: 텍스트 분석 / 검수·라벨링 / 이미지 분석 / 이미지 검색·일괄.
※ 스팸·NER 가중치는 패키지에 포함되어 바로 실행됨 (없으면 위 1)·train_text.py, rnd-detection-models의 finetune_ner.py로 생성).

## 6) 실험 기록 (MLflow)
```bash
mlflow ui --backend-store-uri sqlite:///mlflow.db --port 5000   # 각 export 폴더에서
```

## 데이터 준비 주의
- 공개 데이터(스팸 ko 3분할·coco128 등)는 용량·저작권으로 저장소에 없을 수 있음 → 첫 실행 시 자동 다운로드되거나, 각 폴더 안내대로 준비.
- **직접 라벨링·검수 데이터는 재생성 불가 자산** — 민감 이미지는 로컬에서만 다룸(웹 라벨링 도구 금지).
