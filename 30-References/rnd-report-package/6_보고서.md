## 2. 목표 및 목적

- **목표**: 플랫폼의 **1차 탐지 계층**(텍스트 분류·PII·NER·이미지 탐지·상황 판단)을 공개 데이터로 로컬에서 end-to-end 재현하고, 하나의 데모로 통합한다.
- **목적**: 특정 도메인이 아니라 **도메인 교체 가능한 범용 계층**을 확보 — 아동·청소년 성착취물 대응, 불법스포츠도박 등 복수 적용처에 데이터·프롬프트·룰셋만 바꿔 이식. 실전 데이터 확보 전에 파이프라인·방법론·판단 기준을 근거와 함께 확정해 시행착오 비용을 제거.

## 3. Why · How · What

**Why (왜 필요한가)**
- 탐지 대상 도메인은 여럿이고 계속 늘어난다. 도메인마다 코드를 새로 짜면 반복 비용이 누적 → **규약(계약) 위에서 코드는 고정, 데이터·설정만 교체**하는 구조가 필요.
- 좋은 분류가 무엇인지, 어떤 방법을 언제 쓰는지가 근거 없이 정해지면 실전에서 흔들린다 → 동일 잣대 비교로 방법 선택 기준을 확정.

**How (어떻게 검증했나)**
- 고정 test셋 + 산출물 3종 세트 규약 + MLflow 기록으로 **동일 잣대 비교** 인프라 구축(지시 1).
- 테스트 누출 금지 3단(train으로 제작 / validation으로 조정 / test로 채점만)을 전 실험에 적용.
- 각 모듈을 미니 데이터로 완주 → 통합 데모로 조립 → 사용자 검증.

**What (무엇을 얻었나)**
- 6개 1차 탐지 모듈 + 이를 조립한 데모(탭 4개) + 도메인 무관 판단 기준(가이드).
- 분류·CLIP·YOLO의 실측 성과와, "단일 승자 없음 → 업무 요구로 선택"이라는 방법론.

## 4. 수행 과정

전체 아키텍처·모듈별 구조·데이터 순환 도식은 **[5. 도식도]** 참조. 지시별 수행 요지:

- **지시 1 (산출물 환경)**: `text,label` CSV만 주면 열·클래스 자동 인식 → 학습 → 산출물 3종 세트(가중치+label_map+meta). 데이터셋(정본) → 학습 → 재사용 → 검수 → 편입의 순환 확립.
- **지시 2 (분류)**: 같은 스팸 test셋 300건에서 RULE·BERT·하이브리드 동일 잣대 비교. 오답 교집합 분석으로 앙상블 근거 도출.
- **지시 4 (CLIP)**: zero-shot → linear probe → KoCLIP 3단계. 프롬프트가 곧 분류 체계임을 실증(재학습 없이 교체).
- **지시 3 (통합 UXUI)**: 완료 모듈 5개를 신규 학습 0으로 Gradio 탭 4개에 통합. 검수·라벨링 탭으로 데이터 순환을 UI로 폐쇄.
- **지시 4′ (직접 라벨링)**: labelme(로컬 도구)로 직접 라벨링 → YOLO 규약 변환 → 전이학습 → 탐지 실증.

## 5. 수행 결과 (성과 지표)

**분류 (지시 2, 고정 test 300건)**

| 방법 | 정확도 | 스팸P | 스팸R | F1 |
|---|---|---|---|---|
| RULE | 0.900 | 0.948 | 0.848 | 0.895 |
| BERT-full | 0.953 | 0.993 | 0.914 | **0.952** |
| HYBRID-AND | 0.913 | **1.000** | 0.828 | 0.906 |
| HYBRID-OR | 0.940 | 0.946 | **0.934** | 0.940 |

**CLIP (지시 4, 고정 test 27장)**: 부정형 프롬프트 0.444 / 긍정형 0.778 / **KoCLIP+한국어 0.852(최고)** / 영어모델+한국어 0.519(붕괴).

**YOLO 직접 라벨링 (지시 4′)**: 웹캠 16장 라벨링→학습 → 검증 mAP50 **0.995**, 데모 탐지 확인. 확장 실험(COCO+신규클래스 병합): 신규 클래스 습득했으나 COCO 붕괴(0.177) — **파국적 망각** 실증.

**판독**
- 분류는 **단일 승자 없음** — 오탐 교집합 0(완전 상보)이라 AND에서 오탐 전멸(P 1.0), OR에서 미탐 최소. 업무의 치명적 실수로 방법을 고른다.
- CLIP은 프롬프트 설계가 성능을 좌우(하이퍼파라미터) — 후보 목록 = 탐지 범위의 정본.
- 소데이터 병합 재학습은 파국적 망각 위험 → **범용+특화 모델 병행 운용**이 실무 정답.

## 6. 기대 효과

1. **도메인 즉시 이식**: 새 도메인은 데이터·프롬프트·룰셋만 교체 — 코드 재작성 없이 탐지 계층 재사용.
2. **방법 선택의 근거화**: 오탐/미탐 중 무엇이 치명적인지로 방법(단일·하이브리드)을 고르는 기준 확정.
3. **데이터 자산 순환**: 검수 UI로 모델 판정을 라벨 자산으로 승격 — 쓸수록 기준이 성장.
4. **비용 효율·재현성**: 전 과정 로컬, MLflow 기록으로 재현 가능. 상사 보고 패키지(문서+코드+가중치) 완비.

## 8. 향후계획

1. **실전 도메인 데이터 적용** (핵심 병목): 도메인 라벨·이미지 확보 시 동일 파이프라인으로 즉시 이식 — 룰 키워드 재선정·CLIP 프롬프트 재확정.
2. **OCR 계층 (지시 3.1)**: PaddleOCR·EasyOCR로 이미지→텍스트 추출, 분류·NER 입력으로 연결.
3. **검수 자산 축적 → 재학습**: 데모 검수 탭으로 도메인 라벨을 모아 고정 test셋 위에서 성능 개선 추적.
4. **후속 계층 연동**: 1차 탐지 결과를 Hybrid RAG(근거 검색)·Graph RAG(관계 분석)로 이어 붙이는 상위 파이프라인.
