# BERT vs GPT-2 — 텍스트 분류 비교 (학습 정리)

> 두 트랙(교재 GPT-2 6장 학습 · 업무 BERT R&D)을 같은 "스팸 분류"로 나란히 실습하며 익힌 비교 정리.
> 정본: 이 노트. 관련 — [[../10-Projects/llm-from-scratch/llm-ch6-classify|GPT-2 6장 노트]] · [[rnd-bert-labeling-test-plan|BERT R&D]]

## 0. 트랜스포머 — 둘의 공통 뿌리

- **트랜스포머(Transformer, 2017 "Attention Is All You Need")** = 어텐션을 핵심 부품으로, 문장 속 단어들이 **서로를 동시에 참고**하게 만든 신경망. RNN(순차·먼 단어 잊음)과 달리 **전부 병렬 처리 + 거리 무관 연결**.
- 블록 1개 구조(4장에서 구현): `LayerNorm→멀티헤드 어텐션→드롭아웃→(+잔차)` → `LayerNorm→피드포워드→드롭아웃→(+잔차)`. 이 블록을 **여러 층 쌓음**(GPT-2=12층).
- 부품: 어텐션(단어끼리 정보 교환 Q·K·V) / 피드포워드(단어별 개별 가공) / LayerNorm(값 안정화) / **잔차연결**.
- 원래 트랜스포머 = **인코더 + 디코더**. 후예는 한쪽만 씀: **BERT=인코더**, **GPT=디코더**. → 아래 모든 차이가 여기서 갈림.

### 잔차연결 (residual connection)
- 영어: **residual connection = skip connection = shortcut connection** (셋 다 동의어, ResNet 2015 유래).
- 정의: `출력 = x + 가공(x)`. 입력 x를 가공 결과에 **그대로 더하는 지름길**. (코드 변수명 흔히 `shortcut`)
- **층 쌓기와 별개**: 층 쌓기=세로(깊이↑), 잔차연결=한 블록 안에서 입력을 출력에 더함.
- 동기 = **기울기 소실 해결**: 미분하면 `1 + F'(x)` → 항상 "1"이 보장돼 기울기가 아래층까지 살아 전달. 깊게(12·96층) 쌓아도 학습 가능. (부가: 불필요한 층을 항등 통과로 만들기 쉬움)

## 1. 핵심 차이 한눈에

| | GPT-2 (교재 트랙) | BERT (업무 R&D 트랙) |
|---|---|---|
| 트랜스포머 부분 | **디코더**만 | **인코더**만 |
| 어텐션 방향 | **causal**(왼→오, 단방향) | **양방향**(전체 동시) |
| 태생 목적 | **생성**(다음 단어) → 분류로 전용 | 처음부터 **이해·분류** |
| 분류에 쓰는 출력 | **마지막 토큰** `model(x)[:, -1, :]` | **`[CLS]` 토큰** 벡터 |
| 분류 head | 768→50257 교체 → 768→**2** | `[CLS]` 위 분류층(num_labels=2) |
| 토크나이저 | tiktoken `gpt2` (pad=50256 재활용) | BertTokenizer (`[CLS]`/`[SEP]`/`[PAD]` 전용) |
| 한국어 | ✗ (영어 바이트·영어 데이터) | ✓ (klue/bert-base) |
| 가중치 로드 | 직접 구축 + OpenAI(HF 어댑터로 로컬) | `from_pretrained`(HF) 한 줄 |
| 결과(SMS) | test ~95.7% | test ~96~97% |

## 2. 왜 분류 위치가 다른가 (가장 본질)

- **GPT-2 = causal**: 각 토큰은 자기 **앞쪽만** 봄 → 문장 전체를 본 건 **맨 마지막 토큰**뿐 → 마지막 토큰 출력을 분류에 씀.
- **BERT = 양방향**: 모든 토큰이 서로 봄 → 문장 맨 앞 **`[CLS]`**(의미 없는 빈 토큰)가 전체 요약을 모으도록 학습됨 → `[CLS]` 벡터를 분류에 씀.
- **같은 목적**(문장 1개를 대표하는 벡터 1개 뽑기), **다른 위치**(어텐션 방향 차이 때문).

### `[CLS]` (BERT 전용)
- "Classification". BERT가 **모든 문장 맨 앞에 자동 삽입**하는 특수 토큰. `[SEP]`=문장 끝/구분.
- 자기 의미가 없어 학습 중 **문장 전체 요약 벡터** 역할을 맡음 (회의 서기 비유).
- `BertForSequenceClassification`이 `[CLS]` 추출→분류층 통과를 **내부 자동** 처리.

## 3. 데이터 파이프라인 차이 — `SpamDataset` 두 판

| | GPT-2판 (`data_loader.py`) | BERT판 (`finetune_bert_spam.py`) |
|---|---|---|
| 토크나이즈 시점 | `__init__`에서 전부 미리 | `__getitem__`에서 1건씩(lazy) |
| max_length | 데이터에서 **동적 계산**(train 최장) | 상수 **128 고정**(초과 시 truncation) |
| 패딩 토큰 | `50256`(`<|endoftext|>` 재활용) | 전용 `[PAD]` |
| 패딩 방식 | 손으로 리스트 연결 | `padding="max_length"` 옵션 |
| **attention_mask** | **없음** | **있음**(1=실토큰/0=패딩, 패딩 가림) |
| 반환형 | **튜플** `(토큰, 라벨)` | **dict** 3키 |

- **attention_mask가 BERT에만 있는 이유**: 양방향이라 패딩까지 다 봐버림 → 0으로 가려야 `[CLS]` 요약이 패딩에 오염 안 됨. GPT-2는 causal+마지막 토큰이라 교재가 생략.
- tiktoken은 인코딩 전용(가벼움)이라 "손 패딩", BertTokenizer는 패딩·자르기·mask를 한 번에(풍부).

## 4. 왜 두 트랙으로 나뉘었나

- **나눈 게 아니라 출발점이 다른 두 일**이 같은 소재(스팸 분류)로 만남: GPT-2=교재 6장 학습, BERT=업무 R&D 요청.
- 기술적으로도 아키텍처(인코더/디코더)·어텐션 방향·토크나이저·언어가 전부 달라 자연히 분리.
- 서로의 가치: GPT-2로 "생성모델을 분류로 전용하는 법", BERT로 "분류 전용 모델"을 배워 **두 분류 패러다임 비교**. 한국어 필요 → BERT klue가 담당.

## 5. 공유하는 것

- **데이터 준비 코드**(`dataset_finetuning.py`): 다운로드→라벨 인코딩→클래스 균형화→70:10:20 분할. 두 트랙 공용.
- **개념**: 라벨링↔분류(동전의 양면), 지도학습, train/val/test, 정확도 vs 크로스 엔트로피(대리손실), 패딩으로 길이 통일.
- **로컬 실행**: 둘 다 M4 Max/MPS에서 TF 없이 완주(BERT=HF PyTorch, GPT-2=`hf_weight_adapter.py`).
