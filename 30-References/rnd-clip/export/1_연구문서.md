# 1. 연구문서 — CLIP 이미지 상황 판단 (2026-07-09~10)

## 1. 배경과 목적

상사 지시 4번 "CLIP 이미지 상황 판단". 통합 UXUI(지시 3)의 이미지 탭에 "YOLO 박스 +
상황 태그"를 싣기 위한 선행 R&D — **UI는 있는 모듈을 보여주는 껍데기이므로 모듈이 먼저**.

검증 질문 3개: ① CLIP zero-shot이 실사 이미지의 상황을 판단하는가 ② 소량 라벨 학습(linear
probe)이 zero-shot보다 나은가 ③ 한국어 프롬프트가 가능한가(KoCLIP).

## 2. CLIP의 성격 — 기존 분류기와 무엇이 다른가

CLIP은 이미지와 텍스트를 **같은 좌표계(임베딩 공간)**에 놓도록 4억 쌍의 이미지-캡션으로
사전학습된 모델. 분류가 아니라 "이미지에서 가장 가까운 문장 고르기"로 동작한다.

결정적 차이 = **분류 체계가 사는 곳**: BERT는 체계가 가중치(분류층+label_map)에 박혀
변경에 재학습이 필요하지만, CLIP은 체계가 **입력 문자열(프롬프트)**이라 교체 즉시 반영된다.
실측: 같은 8장을 6후보 상황 체계 → 3후보 실내외 체계로 5초 만에 재분류 (재학습 0).

## 3. 실험 설계

- 데이터: coco128 실사 128장. 사람판별 과제는 YOLO 박스 라벨에서 이미지 단위 라벨 유도
  (person 박스 존재 여부 — 61/67 균형). **기존 자산의 라벨을 새 과제 라벨로 재활용.**
- 분할 70:10:20 (seed 123) — test 27장은 최종 채점 전용, 모든 방법을 같은 27장으로 비교
  (지시 2에서 확립한 동일 잣대 원칙).
- 기록: MLflow 실험 `CLIP-제로샷`(데모)·`CLIP-사람판별-비교`(본 실험, run 6개).

## 4. 결과 (전부 로컬 실측)

### 4-1. zero-shot 상황 판정 데모 (1단계)
coco128 8장 × 상황 후보 6개: 음식 98.1% · 동물 99.7% · 사람들 83~91% — **학습 0줄로 판정**.

### 4-2. 사람판별 — 같은 test 27장 동일 잣대 (2·3단계)

| 방법 | 정확도 | P | R | F1 | 정탐/오탐/미탐 |
|---|---|---|---|---|---|
| zero-shot 부정형("no people") | 0.4444 | 0.417 | 0.385 | 0.400 | 5/7/8 |
| zero-shot 긍정형(대비쌍) | 0.7778 | 0.769 | 0.769 | 0.769 | 10/3/3 |
| linear probe (89장 학습) | 0.7778 | **0.889** | 0.615 | 0.727 | 8/1/5 |
| 영어 모델 + 한국어 프롬프트 | 0.5185 | 0.500 | 0.692 | 0.581 | 9/9/4 |
| **KoCLIP + 한국어 프롬프트** | **0.8519** | **0.909** | 0.769 | **0.833** | 10/1/3 |
| KoCLIP + 영어 프롬프트 | 0.7778 | 0.818 | 0.692 | 0.750 | 9/2/4 |

## 5. 발견 4가지

1. **프롬프트 = 하이퍼파라미터.** 부정형 프롬프트는 동전 이하(0.4444) — CLIP의 알려진
   부정형(negation) 약점이 실측으로 재현. 긍정형 대비쌍으로 재작성하자 0.7778.
   zero-shot의 성능 변수는 학습이 아니라 **문구 설계**다 (룰의 문턱과 같은 지위).
2. **잘 쓴 프롬프트(라벨 0장) ≈ 89장 linear probe.** 정확도 동률(0.7778), 단 오답
   프로필이 다름 — probe는 정밀형(오탐 1), zero-shot은 균형형(3/3). "같은 점수, 다른
   실패 방식" (지시 2 교훈의 재현).
3. **프롬프트 언어는 전제 조건.** 영어 CLIP에 한국어 프롬프트 → 0.5185로 붕괴 (텍스트
   인코더가 한국어를 못 읽음). **KoCLIP + 한국어 = 0.8519 전체 최고** (P 0.909).
   같은 구조라 `BASE=` 교체만으로 기존 스크립트 호환 (계약 설계의 이득).
4. **linear probe = 몸통+머리 패턴의 최소판.** 얼린 임베딩(512차원) 위 선형 머리
   **1,026눈금**만 학습(0.9초) — BERT 분류층(약 1만)보다 작다. 산출물은 지시 1 규약대로
   3종 세트(head.pt + label_map + meta) 저장.

## 6. 결론 — UXUI로 넘기는 확정 근거

**이미지 탭 상황 판단 = KoCLIP(`Bingsu/clip-vit-base-patch32-ko`) + 한국어 긍정형
프롬프트.** 프롬프트 설계 수칙: 부정형 금지 · 대비쌍(찾는 것 vs 그 외의 긍정 묘사) ·
받이 클래스 포함(닫힌 집합 — 어느 후보에도 안 맞으면 억지 배정됨).

## 7. 한계와 다음 단계

- test 27장 — 수치는 잠정 (1장 = 3.7%p). 단 0.52 vs 0.85 격차는 방향성 결론에 충분.
- coco128은 일상 사진 — **도박 도메인 프롬프트·이미지는 미검증** (UXUI 단계에서 자체
  이미지로 재검).
- transformers 5.x 함정: `get_image_features`가 텐서 대신 출력 객체 반환 → `pooler_output`
  사용 (코드에 주석).
- linear probe의 R(0.615) 열세 — 작은 배경 인물 미탐 추정. 라벨 증가 시 재실험 여지.
