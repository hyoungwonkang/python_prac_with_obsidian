# 2. 소스코드 — CLIP 이미지 상황 판단 (2026-07-10)

파일 2개. 실행 환경 `~/rnd-env` (Python 3.12, transformers 5.x).

## clip_zeroshot.py — zero-shot 상황 판정 (1단계)

```
PROMPTS  ← 상황 후보 문장들 ('|' 구분 환경변수) — 이 목록이 곧 분류 체계
pick_images() ← IMAGES 지정 또는 coco128에서 골고루 N장
CLIPModel.logits_per_image → softmax → 이미지마다 최적 후보와 확률 출력
MLflow: 실험 'CLIP-제로샷' (프롬프트·판정 결과 artifact)
```

설계 결정:
- **분류 체계를 환경변수로** — 프롬프트 교체 = 체계 교체가 CLIP의 본질이므로 계약도 그렇게.
- 받이 후보("a photo of people in daily life")를 기본 포함 — 닫힌 집합의 억지 배정 방지.

## clip_linear_probe.py — zero-shot vs 소량학습 비교 (2단계)

```
load_labeled_images() ← coco128 YOLO 라벨 → 이미지 단위 2클래스 (person 박스 유무)
split()               ← 70:10:20 seed 123 (기존 R&D와 동일 규율)
embed()               ← 얼린 인코더로 512차원 임베딩 (L2 정규화)
                        ※ transformers 5.x: get_image_features → .pooler_output
선형 머리 512→2 (1,026눈금) — 전체 89장 = 1배치, 검증 최고 시점의 머리 보관
report()              ← 정확도/P/R/F1 + 네 칸 → MLflow '사람판별-비교'
산출물: artifacts/clip-person-probe/ 3종 세트 (지시 1 규약)
```

설계 결정:
- **zero-shot과 probe를 같은 스크립트에서 같은 test로** — 비교가 목적이므로 잣대 분리 금지.
- `ZS_PROMPTS` 환경변수 — 프롬프트 문구 실험(부정형/긍정형)을 코드 수정 없이.
- 산출물 head.pt는 머리만 저장 — 재사용 시 meta.json의 base_model로 몸통을 로드해 얹는다
  (가중치 422MB 대신 5KB — 몸통은 공용이므로).
