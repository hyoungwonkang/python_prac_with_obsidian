## CLIP의 구조

### 1. 이미지와 텍스트를 같은 좌표계에 — zero-shot의 원리
- CLIP = 4억 쌍의 이미지-캡션으로 "짝이 맞는 것끼리 가깝게" 사전학습된 이중 인코더.
  분류가 아니라 **"이미지에서 가장 가까운 문장 고르기"** — 그래서 클래스가 가중치가 아니라
  입력(프롬프트)에 산다.
- 실측: 학습 0줄로 실사 8장 상황 판정 (음식 98.1%·동물 99.7%), 프롬프트 교체로
  분류 체계 전환 5초 (6후보 상황 → 3후보 실내외).

### 2. 분류 체계란 — 그리고 "어디에 사는가"
- **분류 체계 = 분류기가 고를 수 있는 답안 선택지의 전체 목록** (객관식 보기 구성).
- 사는 곳이 방식을 가른다: 룰=조건문 / BERT=가중치(분류층+label_map) / ko-pii=라이브러리
  코드 / **CLIP=입력 문자열**. 체계 변경 비용 = 코드 수정 / 재학습 / 추출 재실행 / 문자열 교체.
- **체계 변경 빈도가 방식 선택의 숨은 기준** — 자주 바뀌면 CLIP류, 고정+정확도 우선이면 학습형.

## 실측에서 얻은 수칙

### 3. 프롬프트 = 하이퍼파라미터 (zero-shot의 설계 변수)
- 같은 모델·같은 이미지에서 문구만으로 0.4444 ↔ 0.7778. 룰의 문턱, 모델의 LR과 같은
  지위의 조정 변수 — "zero-shot이라 설계할 게 없다"는 착각 금지.
- **CLIP은 부정형을 못 읽는다**: "no people" 류 → 동전 이하. 대비쌍(찾는 것 vs 그 외의
  긍정 묘사)으로 쓸 것. 캐치올 클래스 포함(닫힌 집합 억지 배정 방지 — 36.61% 교훈 동일).

### 4. 잘 쓴 프롬프트(라벨 0장) ≈ 소량 linear probe — 단 오답 프로필이 다름
- 실측 동률(0.7778): probe = 정밀형(P 0.889, 오탐 1) / zero-shot 긍정형 = 균형형(3/3).
  "같은 점수, 다른 실패 방식"(지시 2 교훈)의 재현 — 용도가 P·R 요구에 따라 갈림.
- 순서 수칙: **probe(라벨링) 전에 프롬프트 개선부터 소진** — 문구 수정이 라벨링보다 싸다.

### 5. 프롬프트 언어는 전제 조건
- 영어 CLIP + 한국어 프롬프트 = 0.5185 붕괴 (텍스트 인코더가 한국어를 못 읽음 — 토큰이
  학습 분포 밖). **KoCLIP + 한국어 = 0.8519 전체 최고** (P 0.909).
- 같은 CLIP 구조면 `BASE=` 교체만으로 스크립트 호환 — 환경변수 계약 설계가 모델 교체에서도
  값을 함 (지시 1 철학의 연장).

### 6. linear probe = 몸통+머리 패턴의 최소판
- 얼린 임베딩(512차원) 위 선형 머리 **1,026눈금**·0.9초 — BERT 분류층(약 1만)보다 작다.
  파인튜닝 스펙트럼: 풀 파인튜닝(전체) > 분류층 학습(BERT R&D) > **linear probe(임베딩 고정)**.
- 산출물은 머리만 저장(head.pt 5KB) — 몸통은 공용이므로 meta의 base_model로 재조립.
  3종 세트 규약은 동일 (지시 1).

### 7. 라벨 유도 — 기존 자산에서 새 과제 라벨을 공짜로
- coco128의 YOLO 박스 라벨(객체 위치용) → "person 박스가 있는가"로 이미지 단위 2클래스
  유도. **라벨은 목적에 맞게 변환 가능한 자산** — 새 라벨링 없이 새 과제가 생겼다.
- 유도가 유효한 근거 = **정보의 방향**: 박스 라벨(상세)→이미지 라벨(요약)은 읽기만 하면 되는
  변환(추측 0), 역방향(요약→상세)은 불가. 비유: 사진마다 붙은 물건 목록표를 읽어 "사람
  있음/없음" 스티커를 붙이는 일. 같은 패턴: NER 개체 목록 → "개인정보 포함 여부" 문서 라벨.
- 유도 후 분포 확인은 필수 — 61/67 균형이라 accuracy가 유효했음 (123:5였다면 "전부 없음"
  모델이 96점 — 시험 무효).

### 8. transformers 5.x 실전 함정 (버전 마이그레이션)
- `get_image_features()`가 4.x의 텐서 대신 **출력 객체**를 반환 — 512차원은 `.pooler_output`.
  같은 코드가 버전에 따라 AttributeError로 죽는 형태. 라이브러리 메이저 업그레이드 시
  반환형부터 확인.

### 9. seed = 재현 가능한 무작위
- 컴퓨터의 무작위는 **시작값(seed)이 같으면 항상 같은 순서로 섞이는 가짜 무작위** —
  seed 고정 = 언제 어디서 다시 돌려도 정확히 같은 test 27장 = 실험 재현성.
- 숫자(123) 자체는 무의미 — "고정했다"는 사실과 **vault 관례와의 일치**(스팸 R&D부터 123)가
  중요. 관례가 같으면 "분할이 달라서 점수가 달라졌나"라는 의심 하나가 통째로 사라진다.

### 10. 누출 경계선 자기 평가 — 프롬프트를 test에서 골랐다 (2026-07-10 사후 발견)
- 프롬프트 3종(부정형/긍정형/단순명사)을 **test 27장에서 비교해 좋은 것을 채택** —
  "프롬프트 = 하이퍼파라미터"인데 하이퍼파라미터를 test로 고른 셈. 지시 2의 룰 문턱
  (validation으로 조정)과 대비되는 **누출 금지 3단의 경계선 위반**.
- 처방(연구문서 한계와 연결): UXUI 실전 프롬프트는 자체 이미지(새 데이터)에서 재확정.
  "27장 잠정" 단서의 숨은 이유 중 하나.
- 교훈: 원칙은 세워도 밟게 된다 — **어디를 밟았는지 알고 기록하는 것**까지가 방법론.
  발견 자체(부정형 약점)는 유효하되, 그 발견으로 "고른 값"은 새 데이터 검증 전까지 잠정.
