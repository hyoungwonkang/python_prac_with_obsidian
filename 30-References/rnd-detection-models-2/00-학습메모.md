## OpenCV (2026-07-08)

### 역할 — 학습이 아니라 전처리·비식별
- OpenCV는 파인튜닝 대상이 아니다. 아키텍처 **②추출·전처리** 계층의 유틸리티 — 수집 원본을 ③ 탐지에 넣기 전 규격화하고, 인물을 비식별한다.
- 4기능: 기본처리(로드·그레이·리사이즈·블러·엣지) / 영상 프레임 추출 / 비식별화 / end-to-end 파이프라인.

### ★ cv2 5.0 빌드에 objdetect(Haar)가 없다
- `opencv-python==5.0.0`에서 `cv2.CascadeClassifier`·`cv2.objdetect`가 **존재하지 않음**. `cv2.data.haarcascades` 폴더도 빈 상태.
- 교훈: "OpenCV = 얼굴검출"이라는 통념이 빌드에 따라 깨진다. 버전 고정·기능 존재 확인이 먼저.
- 우회: **검출은 YOLO(사전학습), 비식별 처리는 OpenCV**로 분담. 배치1 [[../rnd-detection-models/00-학습메모|YOLO]] 산출물을 그대로 재활용 → 두 R&D가 파이프라인으로 연결됨.

### 비식별화 = "검출 박스 → 그 영역만 블러/모자이크"
- `blur_roi`: 박스 좌표로 ROI를 잘라 처리 후 되붙임.
  - 블러 = `GaussianBlur`(커널은 **홀수·양수** 필수, 영역 크기 비례).
  - 모자이크 = 1/16 축소 → `INTER_NEAREST` 확대.
- **비식별 성공의 정량 근거**: 라플라시안 분산(=선명도)이 원본 대비 급감. 실측 zidane 100.7 → 13.8(−86%). 육안이 아니라 수치로 증명하는 습관(배치1 F1·mAP과 같은 결).

### 영상 ↔ 프레임
- `VideoWriter`(fourcc `mp4v`)로 합성 영상 생성 → `VideoCapture`로 되읽어 `EVERY`프레임마다 저장. 외부 영상 없이도 프레임 추출 로직을 자립 검증.
- 동영상 증적 → 프레임 이미지 = 이미지 분석(③) 입력 확보의 전제.

### 잔 함정
- `imread`는 실패 시 예외 없이 **None** 반환(경로 오타·HTML·손상) → 로드 직후 확인.
- OpenCV 색 순서는 **BGR**. RGB 기대 라이브러리로 넘길 때 `cvtColor(BGR2RGB)`.

### 비식별 심화 — 사람용/기계용 두 갈래 (2026-07-14, OCR 착수 전 Q&A에서 확정)
- **비식별 = 신원 단서만 제거하고 쓸모는 남기는 처리** — 개념이고, 블러(이미지)·토큰화(텍스트,
  PII)는 데이터별 구현. 공통 2단: 찾아서(YOLO 검출/PII 탐지) → 가린다(블러/토큰).
- 가역성이 급을 가른다: 블러 = 비가역(정보 파괴) ↔ PII Vault = 가역(금고 보관, 권한자만 복원).
  원본 동결 저장(SHA-256·Object Lock)이 별도로 필요한 이유.
- **비식별본은 ③판정에 합류하지 않는다** — 선명도 −86%는 기계도 못 읽는다는 뜻. 기계 분석
  (OCR·③판정)은 동결 원본에서, 비식별본은 사람 눈(검수 UI·공유) 전용. 두 갈래 병렬 산출물.
- 비식별의 YOLO는 ③판정이 아니라 **②유틸(비식별의 눈)** — 같은 가중치, 다른 역할.
  블러는 좌표가 필요하므로 검출이 반드시 선행.
- 선명도 수치(100.7→13.8)는 보관 데이터가 아니라 **방법 검증 기록** — 운영화하면 감사 로그·
  품질 게이트(블러 누락 유출 방지)로 승격 가능.
