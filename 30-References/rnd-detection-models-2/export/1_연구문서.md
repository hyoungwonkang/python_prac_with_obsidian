## 공통 배경 — 이 배치가 아키텍처의 어디인가

플랫폼 탐지 흐름: `① 수집 → ② OCR/추출·전처리 → ③ 1차 탐지(Rule·분류·NER·이미지) → ④ 근거검색 → ⑤ 관계분석 → ⑥ 정형판단 → ⑦ 위험도`

| 구성요소 | 계층(흐름) | 성격 |
|---|---|---|
| **OpenCV** | ② 추출·전처리 | 라이브러리 실습 (학습 없음) |
| RULE | ③ 1차 탐지 Rule Engine | 룰 직접 구현 (학습 없음) |
| CLIP(VLM) | ③ Image Analyzer | 유일한 파인튜닝(linear probe) |
| PaddleOCR | ② 이미지→텍스트 | 라이브러리 추론 |

## A. OpenCV — 이미지·영상 전처리 및 비식별화

### A-1. 목표 및 목적

- **목표**: 수집된 이미지/동영상 증적을 1차 탐지(YOLO·이미지 분석)에 넣기 전 **전처리**하고, 개인정보 보호를 위한 **비식별화**를 코드로 재현한다.
- **목적**: 아키텍처 ②추출·전처리 계층과 개인정보 필수사항("민감정보 최소 수집·비식별")을 실제 동작으로 확보. 파인튜닝이 아니라 **라이브러리 기본기 검증**.

### A-2. Why · How · What

**Why** — 수집 원본은 해상도·형식이 제각각이고 인물이 포함될 수 있다. ③ 탐지에 넘기기 전 **규격화(리사이즈)·노이즈 완화·비식별**이 필요하며, 동영상 증적은 **프레임 단위 이미지**로 분해해야 이미지 분석이 가능하다.

**How** — OpenCV(cv2) 라이브러리로 4개 최소 기능을 구현·검증:
1. 기본 처리 — 로드·그레이스케일·리사이즈(640 기준)·가우시안 블러·캐니 엣지
2. 영상 프레임 추출 — `VideoCapture`로 N프레임마다 저장
3. 비식별화 — 인물 영역만 흐리게(가우시안/모자이크)
4. end-to-end 파이프라인 — 프레임 → 검출 → 비식별 → 저장

**중요한 기술 결정 (환경 제약 대응)**: 이번 `opencv-python==5.0.0` 빌드에는 **`objdetect` 모듈(Haar `CascadeClassifier`)이 포함돼 있지 않다.** 따라서 얼굴 검출을 OpenCV 단독으로 할 수 없다. → **검출은 이미 검증한 YOLO(사전학습)로, 비식별 처리는 OpenCV로** 역할을 나눴다. 이는 배치1 YOLO R&D를 그대로 재활용하는 자연스러운 결합이기도 하다.

**What** — 어떤 환경에서도 실행되는 자립형 스크립트 4종(외부 이미지 없으면 ultralytics 번들 샘플/합성 이미지·영상 자동 사용)과, 비식별이 실제로 정보를 지운다는 정량 근거.

### A-3. 처리 흐름

```
[동영상/이미지 수집]
      │  opencv_frames.py (VideoCapture)
      ▼
[프레임 이미지]  ──opencv_basics.py(리사이즈·정규화)──►  [YOLO 입력 규격]
      │  opencv_pipeline.py
      ▼
[YOLO 사람 검출]  ──박스 좌표──►  [opencv_deidentify.blur_roi: 영역 블러/모자이크]
      ▼
[비식별 처리된 이미지 저장]  →  ③ 이미지 분석·채증 보존
```

### A-4. 수행 결과 (실측)

| 스크립트 | 입력 | 결과 |
|---|---|---|
| `opencv_basics.py` | bus.jpg (810×1080) | 640×853 리사이즈, 캐니 엣지 픽셀 146,823개 |
| `opencv_frames.py` | 합성 영상 60프레임(30fps) | EVERY=5 → **12장** 추출 |
| `opencv_deidentify.py` | zidane.jpg | 사람 **2명** 검출·블러, **선명도 100.7 → 13.8(−86%)** |
| `opencv_pipeline.py` | 샘플 2장(프레임 묶음) | 사람 **6명(4+2)** 검출·비식별 |

**판독**
- 비식별화가 실제로 정보를 지운다는 근거 = **라플라시안 분산(선명도) 86% 감소** + 원본과 픽셀 불일치. 육안이 아니라 수치로 검증.
- OpenCV 단독으로 얼굴 검출은 불가했으나, **YOLO 검출 + OpenCV 블러** 결합으로 인물 비식별 목적을 달성 — 배치1 산출물의 재활용.
- 전처리 4기능이 모두 로컬에서 즉시 동작 → 도메인 이미지/영상이 확보되면 그대로 적용 가능.

### A-5. 기대 효과

1. **전처리 규격화**: 수집 원본 → YOLO/이미지 분석 입력 규격 자동 정렬(리사이즈·색공간·노이즈).
2. **개인정보 보호 준비**: 인물 비식별(블러·모자이크)을 파이프라인에 내장 — 채증 보존 시 개인정보 최소화 요구 충족.
3. **동영상 증적 활용**: 영상 → 프레임 분해로 이미지 분석 대상 확대.
4. **비용 0**: 학습 불필요·전 과정 로컬.

### A-6. 산출물

| 파일 | 역할 |
|---|---|
| `opencv_basics.py` | 로드·그레이·리사이즈·블러·엣지 |
| `opencv_frames.py` | 영상 → 프레임 추출 |
| `opencv_deidentify.py` | YOLO 검출 + OpenCV 블러/모자이크 비식별 |
| `opencv_pipeline.py` | 프레임→검출→비식별 end-to-end |
| `requirements.txt` | 고정 버전 의존성 |

### A-7. 향후 계획

1. 도메인 영상/이미지 확보 시 동일 파이프라인 적용.
2. 얼굴 단위 비식별이 필요하면 얼굴 검출기(DNN 기반, 별도 모델 파일) 도입 검토 — 현 빌드의 objdetect 부재 보완.
3. RULE·CLIP·PaddleOCR 진행 후 본 문서에 절 추가.
