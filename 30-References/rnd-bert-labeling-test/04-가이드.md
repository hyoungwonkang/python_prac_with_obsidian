# 04 · 가이드 — 재현 절차 & 주의점

> R&D 산출물 4/4 (가이드). 처음부터 끝까지 따라 하는 재현 + 함정·확장. 사용법: [[03-사용법]].

## A. 처음부터 재현 (체크리스트)
- [ ] 1. Python 3.9+ 가상환경 + `pip install torch transformers scikit-learn pandas`
- [ ] 2. `dataset_finetuning.py` 실행 → `train/validation/test.csv` 생성 (**라벨링**)
- [ ] 3. `finetune_bert_spam.py` 실행 → 학습 + test 평가 (**BERT + 테스트**)
- [ ] 4. 출력의 `accuracy`·`confusion matrix` 확인 → 97%대면 정상
- [ ] (선택) `EPOCHS`, `MAX_LENGTH`, `LR` 조정해 재실험

## B. 3주제가 어디서 재현되나
| 주제 | 재현 지점 |
|---|---|
| BERT 모델 | `BertForSequenceClassification` 로드 + 학습 로그 |
| 라벨링 방법론 | `dataset_finetuning.py` (인코딩·균형·분할) + CSV |
| 테스트 | `finetune_bert_spam.py` ⑤ 평가 블록 출력 |

## C. 주의점 (함정)
1. **test셋 오염 금지** — test.csv는 학습/튜닝에 절대 사용 X. 한 번만, 마지막에 채점.
2. **클래스 균형** — 원본은 ham 87%로 불균형. 균형화 안 하면 "전부 ham" 찍어도 87% 나와 **정확도가 거짓**이 됨. → 균형 + 정밀도/재현율 같이 볼 것.
3. **언어** — `bert-base-uncased`는 **영어 전용**. 한국어 데이터엔 부적합.
4. **MAX_LENGTH=128** — 긴 문장은 잘림. SMS엔 충분하나 다른 데이터는 분포 확인.
5. **재현성** — `SEED=123` 고정했으나 MPS/GPU 비결정 연산으로 소수점은 약간 흔들릴 수 있음.
6. **과적합 신호** — train_loss는 계속 내려가는데 val_acc가 정체/하락하면 epoch 줄이기.

## D. 확장
| 목표 | 방법 |
|---|---|
| **한국어** 스팸/리뷰 | `klue/bert-base` + 한국어 데이터(NSMC 등)로 동일 파이프라인 |
| **속도** (큰 데이터/epoch) | Colab GPU 런타임 — 코드 변경 없이 동일 실행 |
| 다른 분류 작업 | 감성분석·주제분류 등 — 데이터만 교체(라벨 2→N은 `num_labels`만 변경) |
| 단어 단위 태깅(NER) | 패턴 B(`BertForTokenClassification`) — 별도 (本 과제는 문장 분류=패턴 A) |
