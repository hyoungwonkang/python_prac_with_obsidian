# 가이드 — 재현 절차 & 주의점

## A. 처음부터 재현 (체크리스트)
- [ ] 1. Python 3.9+ 가상환경 + `pip install torch transformers scikit-learn pandas`
- [ ] 2. `python dataset_finetuning.py` → `train/validation/test.csv` 생성 (라벨링)
- [ ] 3. `python finetune_bert_spam.py` → 학습 + 테스트 평가 (BERT + 테스트)
- [ ] 4. 출력의 `accuracy`·`confusion matrix` 확인 → 97%대면 정상
- [ ] (선택) `EPOCHS`, `MAX_LENGTH`, `LR` 조정해 재실험

## B. R&D 3주제 재현 지점
| 주제 | 재현 위치 |
|---|---|
| BERT 모델 | `BertForSequenceClassification` 로드 + 학습 로그 |
| 라벨링 방법론 | `dataset_finetuning.py` (인코딩·균형화·분할) + 생성된 CSV |
| 테스트 | `finetune_bert_spam.py` ⑤ 평가 블록 출력 |

## C. 주의점 (함정)
1. **테스트셋 오염 금지** — `test.csv`는 학습·튜닝에 절대 사용하지 않는다. 마지막에 한 번만 채점.
2. **클래스 균형** — 원본은 정상 87%로 불균형. 균형화하지 않으면 "전부 정상" 예측만 해도 87%가 나와 정확도가 왜곡된다. → 균형화 + 정밀도/재현율을 함께 본다.
3. **언어** — `bert-base-uncased`는 영어 전용. 한국어 데이터에는 부적합.
4. **MAX_LENGTH=128** — 더 긴 문장은 잘린다. SMS에는 충분하나 다른 데이터는 길이 분포를 확인.
5. **재현성** — 시드(123)를 고정했으나 GPU/MPS의 비결정 연산으로 소수점은 약간 흔들릴 수 있다.
6. **과적합 신호** — train_loss는 계속 내려가는데 val_acc가 정체/하락하면 epoch을 줄인다.

## D. 확장 방향
| 목표 | 방법 |
|---|---|
| 한국어 스팸/리뷰 분류 | `klue/bert-base` + 한국어 데이터로 동일 파이프라인 |
| 속도 향상 (대용량/다 epoch) | GPU 런타임(Colab/CUDA) — 코드 변경 없이 동일 실행 |
| 다른 분류 작업 | 감성분석·주제분류 등 — 데이터만 교체 (클래스 수는 `num_labels` 변경) |
| 단어 단위 태깅(NER) | `BertForTokenClassification` 사용 (본 과제는 문장 단위 분류) |

## E. 검증 환경
- macOS / Apple Silicon (M4 Max) / `torch 2.8.0`(MPS) / `transformers 4.57.6` / `scikit-learn 1.6.1`.
- **TensorFlow 불필요** — Hugging Face의 PyTorch 백엔드만 사용.
- 별도 GPU 없이 로컬에서 3 epoch 학습·평가 전 과정 완료 (수 분 소요).
