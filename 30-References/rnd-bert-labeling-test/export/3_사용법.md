# 사용법 — 실행 방법 (How to Run)

Apple Silicon(MPS) 로컬 환경 기준 실측 검증. GPU(Colab/CUDA)에서도 코드 변경 없이 동일 실행.

## 1. 환경 설치
```bash
# Python 3.9 이상, 가상환경 권장
pip install torch transformers scikit-learn pandas
```
검증 버전: `torch 2.8.0` · `transformers 4.57.6` · `scikit-learn 1.6.1`. TensorFlow는 필요 없음.

## 2. 데이터 준비 (라벨링)
```bash
python dataset_finetuning.py
```
- SMS Spam Collection 다운로드 → 라벨 인코딩(정상=0/스팸=1) → 클래스 균형화 → 학습70/검증10/테스트20 분할.
- 산출물: `train.csv`(1045) · `validation.csv`(149) · `test.csv`(300).

## 3. 학습·평가 실행
```bash
python finetune_bert_spam.py            # 기본 3 epoch
EPOCHS=1 python finetune_bert_spam.py   # 빠른 확인 (1 epoch도 97%대)
```
- 첫 실행 시 `bert-base-uncased` 가중치(약 440MB)가 자동 다운로드된다.
- Apple Silicon에서 일부 연산 호환을 위해 `PYTORCH_ENABLE_MPS_FALLBACK=1`을 앞에 붙이면 안전하다.

## 4. 결과 해석
```
[epoch 3] train_loss 0.0351 | val_acc 0.9933   학습 진행(손실↓) + 중간 점검
=== TEST 결과 ===
accuracy: 0.9767                                최종 성능(처음 보는 300건)
confusion matrix [[148 1] / [6 145]]            행=실제, 열=예측
spam precision 0.9932 recall 0.9603             정상 문자 오차단 거의 없음
```
- train_loss 감소 = 학습 정상 / val_acc = 과적합 점검 / **test accuracy = 실제 성능**.
- 혼동행렬: 좌상=정상 정답, 우하=스팸 정답, 우상=정상→스팸 오탐, 좌하=스팸 미탐.

## 5. 문제 해결
| 증상 | 조치 |
|---|---|
| `No module named transformers` | `pip install transformers` |
| CSV 파일을 찾지 못함 | 2단계(데이터 준비)를 먼저 실행 |
| 실행이 느림 | GPU 사용 또는 EPOCHS 축소 (데이터가 작아 로컬도 수 분 내 완료) |
| Apple Silicon 연산 오류 | `PYTORCH_ENABLE_MPS_FALLBACK=1` 환경변수 추가 |
