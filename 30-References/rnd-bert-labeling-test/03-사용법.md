# 03 · 사용법 — 실행 방법 (How to Run)

> R&D 산출물 3/4 (사용법). 로컬 M4 Max(MPS) 기준 **실측 검증됨**. 코드: [[02-소스코드]].

## 1. 환경 설치
```bash
# Python 3.9+, 가상환경 권장
pip install torch transformers scikit-learn pandas
```
- 검증 버전: `torch 2.8.0` · `transformers 4.57.6` · `scikit-learn 1.6.1`.
- **TensorFlow 불필요** (HF PyTorch 백엔드).

## 2. 데이터 준비 (라벨링)
```bash
cd 10-Projects/llm-from-scratch
python dataset_finetuning.py     # → train.csv / validation.csv / test.csv 생성
```
- SMS Spam Collection 다운로드 → ham/spam→0/1 라벨 → 클래스 균형 → 70:10:20 분할.
- 산출: `train.csv`(1045) `validation.csv`(149) `test.csv`(300).

## 3. 학습·평가 실행
```bash
cd 30-References/rnd-bert-labeling-test
python finetune_bert_spam.py            # 기본 3 epoch
# 에포크 조절:
EPOCHS=1 python finetune_bert_spam.py   # 빠른 확인 (1 epoch도 정확도 97%대)
```
- 첫 실행 시 `bert-base-uncased` 가중치(~440MB) 자동 다운로드.
- MPS 일부 연산 fallback 대비: `PYTORCH_ENABLE_MPS_FALLBACK=1` 붙이면 안전.

## 4. 실행 결과 보는 법
```
[epoch 3] train_loss 0.0351 | val_acc 0.9933   ← 학습 진행(손실↓) + 중간점검
=== TEST 결과 ===
accuracy: 0.9767                                ← 최종 실력(처음 보는 300건)
confusion matrix [[148 1]/[6 145]]             ← 행=실제, 열=예측
spam precision 0.9932 recall 0.9603            ← 정상문자 오차단(FP) 적음
```
- **train_loss↓** = 학습 잘 됨 / **val_acc** = 과적합 점검 / **test accuracy** = 진짜 성능.
- **혼동행렬**: 좌상=ham 정답, 우하=spam 정답, 우상=정상→스팸 오차단(FP), 좌하=스팸 놓침(FN).
