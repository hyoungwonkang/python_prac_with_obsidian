## 0. 한눈에

사전학습된 **BERT**에 분류층을 얹어, **라벨링된** SMS 데이터로 파인튜닝하고, 학습에 쓰지 않은 **test셋**으로 성능을 채점했다. **test 정확도 97.67%** (로컬 M4 Max/MPS, 3 epoch).

---

## 1. BERT 모델

### 무엇인가
문장을 한 단어씩이 아니라 **전체를 동시에** 보고 단어 간 관계(어텐션)를 계산하는 언어 모델. 같은 단어도 문맥으로 의미가 갈리는 걸 잡아낸다. 상세: [[../BERT_학습정리]] §1~§5.

### 왜 스팸분류에 적합한가 — 패턴 A
BERT는 문장 맨 앞 `[CLS]` 토큰에 **문장 전체 요약**이 모이도록 학습돼 있다. 이 `[CLS]` 벡터 하나에 **분류층**만 얹으면 "문장 → 라벨 하나" 작업이 된다. ([[../BERT_학습정리]] §5 패턴 A)

```
[CLS] Free entry win prize ... [SEP]
  │
  └─ [CLS] 벡터(768) → 분류층 → P(spam)=0.99 / P(ham)=0.01
```

- 스팸분류 = **"문장 전체에 라벨 하나(스팸/햄)"** = 텍스트 분류 = 패턴 A.
- 구현: `BertForSequenceClassification(bert-base-uncased, num_labels=2)` — 파라미터 1억(109M).
- **트랜스퍼 러닝**: 무거운 본체(사전학습)는 재사용, 얇은 분류층만 새로 학습.

---

## 2. 라벨링 방법론

분류는 **지도학습** — 정답(라벨)이 있어야 학습된다. 라벨링과 분류는 **동전의 양면**(라벨링=답 달기 / 분류=그 답 맞히기).

### 4단계 (코드: `dataset_finetuning.py`)
| 단계 | 처리 | 코드 |
|---|---|---|
| 원천 라벨 | SMS Spam Collection (사람이 ham/spam 주석) | 데이터셋 제공 |
| **라벨 인코딩** | 텍스트 라벨 → 숫자 (모델 입력형) | `.map({"ham":0,"spam":1})` |
| **클래스 균형화** | ham을 spam 개수로 언더샘플 (편향 방지) | `create_balanced_dataset` |
| **분할** | train 70 / val 10 / test 20 | `random_split(df,0.7,0.1)` |

- 균형화 결과: 원본 ham 4825/spam 747 → **747/747** → 분할 train 1045 / val 149 / test 300.
- **원칙**: 결과 품질은 라이브러리가 아니라 **라벨 데이터의 양·품질**이 좌우 (garbage in, garbage out).

## 3. 테스트

### test셋이란
**학습에 단 한 번도 쓰지 않고** 떼어둔 데이터(여기선 300건/20%). 완성된 모델을 **처음 보는 데이터로 마지막에 한 번** 채점 → 실전(일반화) 성능.

| 세트 | 용도 | 시험 비유 |
|---|---|---|
| train (1045) | 학습 | 문제집 |
| validation (149) | 학습 중 점검 | 모의고사 |
| **test (300)** | **최종 채점 1회** | 본시험 |

→ train으로 채점하면 컨닝(부풀려짐), validation도 튜닝에 써서 간접 오염 → **순수 test셋**이라야 정직.

### 채점 지표
- **정확도(accuracy)** = 맞힌 수 / 전체.
- **혼동행렬(confusion matrix)** = FP(정상→스팸 오차단) / FN(스팸 놓침) 세부.
- **정밀도/재현율** — 스팸분류는 **정상 문자 오차단(FP)** 비용이 커서 spam **정밀도**가 특히 중요.

### 검증 결과 (test.csv 300건, 학습 미사용)
```
accuracy: 0.9767
혼동행렬 [행=실제, 열=예측]      precision  recall   f1
 [[148   1]   ← 실제 ham        ham   0.9610  0.9933  0.9769
  [  6 145]]  ← 실제 spam       spam  0.9932  0.9603  0.9764
```
- ham 149건 중 **1건만** 스팸 오분류(FP) → 정상 문자 차단 거의 없음(정밀도 높음).
- spam 151건 중 6건 놓침(FN, 재현율 0.96).

---

## 4. 결론
사전학습 BERT + 적절한 **라벨링**(인코딩·균형·분할) + 분리된 **test셋** 채점으로, 작은 데이터(train 1045)·짧은 학습(3 epoch)·로컬 환경만으로 **97.67%** 달성. 3주제가 하나의 파이프라인으로 연결됨을 확인.
