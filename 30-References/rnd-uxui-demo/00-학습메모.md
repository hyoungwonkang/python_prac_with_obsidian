## 화면과 표기

### 1. 터미널 통과 ≠ 브라우저 통과 (HTML 이스케이프 함정)
- `<PERSON_1>` 류 토큰이 Markdown 렌더러에서 **HTML 태그로 해석돼 삼켜짐** — 마스킹 문장이
  중간에서 잘리고 토큰 열이 빈칸. 터미널 테스트는 통과했고 브라우저에서만 발현.
- 처방: `<`→`&lt;` 이스케이프. 교훈: **최종 렌더러 위에서의 확인(스크린샷)까지가 검증** —
  실제로 사용자 스크린샷으로 발견된 버그.

### 2. 성능 표기의 의미론 — "현재 가중치의 test셋 실측치"
- 명패(과거 인용) 비유도, 무조건 "현재 성능"도 부정확. **가중치는 불변**이므로 측정치는
  낡은 기록이 아니라 현재 속성 — 단, **측정 잣대(어느 test셋) 명시가 필수** (성능 = 모델×시험지의 짝.
  데모에 들어오는 실전 문장은 그 시험지와 분포가 다름).
- 하드코딩 숫자는 낡는다 — "손으로 베낀 문서는 반드시 낡는다" 원칙의 UI판.
  처방: meta.json 연동 (가중치 교체 → UI 자동 추종). 단 meta에 있는 지표(정확도)로 표기 교체 필요.
- 데모에서 문장을 아무리 분석해도 test 정확도는 불변: 추론은 eval+no_grad(가중치 불변)이고
  입력에는 정답이 없음(두 가족) — 숫자가 바뀌는 유일한 경로는 재학습→새 산출물.

## ko-pii의 내부 구조 (오탐 한 행을 파고들어 발견)

### 3. 탐지·조치·종합의 3층 분리
- **조치(action) 3단**: 확신 높음→TOKENIZE(치환) / 애매(실측 0.55)→**REVIEW**(자동 마스킹 보류,
  사람 검토 회부 — 표의 "-"의 정체) / 미달→무시. 기계가 확실한 건 자동, 애매한 건 사람에게 —
  순환도의 검수 철학과 동일 설계.
- "고객님만"→PERSON 오탐의 원인: **"~님" 호칭 패턴 룰** (조사 '만'까지 문 경계 오류 덤).
  같은 문장에서 학습형 NER은 "개체 없음" 정답 — **룰 vs 학습의 대비가 한 행에서 실연**
  (인명은 NER이 보완한다는 하이브리드 근거의 살아있는 증거).

### 4. 민감도(항목) vs PII 종합 위험도(문장) — 재식별 조합 논리
- **민감도 = 유형의 속성** ("PERSON은 민감한 부류") — 오탐이어도 유형 등급이 붙음.
  **종합 = 문장 판정** ("이 문장의 PII만으로 개인이 특정되는가") — 스팸 등 다른 위험과 무관한 PII 한정.
- 계산 원리: **식별자**(주민번호 — 1개만 있어도 즉시 식별 → CRITICAL 직행) vs
  **준식별자**(이름·전화·계좌 — 결합될수록 계단: 1종 LOW → 2종 MEDIUM).
  실측 극단 사례: 개별 HIGH + 종합 LOW 공존 ("고객님만" 문장) — 두 값이 다른 질문에 답하는 증거.
- 개별/조치/종합 3중 완충 → 오탐 하나가 문서 전체를 차단하지 않는다.

## 데이터 순환의 실전

### 5. 의사 라벨링(pseudo-labeling)의 안전 설계
- 모델 판정을 정답으로 직행시키면: 두 가족 원리 위반 + 오염 실측(둘 다 정상 중 미탐 3.3%)
  + **자기 실수가 굳어짐**(놓치는 유형이 정상 라벨로 재학습) + 쉬운 사례만 남음(경계 사례가 걸러짐).
- 실무 형태: **판정 = 초안, 사람 확정 버튼으로만 정답 승격** (UI 구조로 원칙을 강제).
  검수 예산은 **불일치 사례 우선**(능동 학습). 검수 UI의 출력 계약 = 학습기의 입력 계약
  (text,label CSV → datasets-own/) — 순환도 마지막 화살표의 실체화.

### 6. 소량 데이터의 실패 사슬과 가드
- 1건 → 70:10:20 분할 시 train 0건(int(1×0.7)=0) → 빈 목록이 토크나이저에 → 깊은 곳에서
  IndexError(원인 안 보임). **계약 위반은 조기에, 한국어로** — 빈 분할·단일 클래스 가드 추가.
- 소량 검수분의 정석은 단독 학습이 아니라 **기존 train에 편입** (고정 val·test는 불변 — 잣대 유지).

### 7. 첫 검수 자산 6건의 실측 교훈
- 불일치 사례 3/6 우선 검수(능동 학습 실천), **RULE 초안 뒤집기 3건 = 룰 약점이 정답과 함께
  기록**된 교과서 (문맥맹 오탐 2 + 키워드 미탐 1). BERT 뒤집기 0건 — 모델 약점 지도는 아직.
- 빈 행(`,,,,,`) 사고: label NaN → 클래스 3개('0','1','nan') 오인 위험. **자산 수정도 UI 경로로**
  (편집기 직접 수정 금지) — 검수 탭이 중복·형식을 보증하는 유일한 문.
