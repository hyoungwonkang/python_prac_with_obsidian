# 3. 사용법 — OCR 추출 계층

> 전 명령은 `rnd-ocr/export/`에서 실행. 두 환경을 쓴다:
> **rnd-env**(EasyOCR·하류 모듈) / **ocr-env**(PaddleOCR 격리 — 의존성 충돌 방지).

## 0) 환경 준비 (1회)

```bash
# rnd-env에 EasyOCR 추가 (기존 스택과 무충돌 실측: easyocr 1.7.2)
~/rnd-env/bin/pip install easyocr

# PaddleOCR은 격리 환경에 (paddlepaddle 3.3.1 + paddleocr 3.7.0 실측 검증)
/opt/homebrew/opt/python@3.12/bin/python3.12 -m venv ~/ocr-env
~/ocr-env/bin/pip install paddlepaddle paddleocr
```
- 첫 실행 시 모델 자동 다운로드 (Easy ~150MB → `~/.EasyOCR/`, Paddle PP-OCRv5 → 캐시).
- 격리 이유: Paddle 동반 opencv 4.10·numpy 2.3.5가 rnd-env 고정 버전(5.0/2.5.1)과 충돌.

## 1) 테스트 이미지 생성 (정답 아는 문장 → clean + 열화 4종)

```bash
~/rnd-env/bin/python make_test_images.py                  # → images/ + ground_truth.txt
TEXT="다른 문장" ~/rnd-env/bin/python make_test_images.py  # 문장 교체 (표본 확대)
```

## 2) 채점 (엔진 중립 잣대 — CER)

```bash
~/rnd-env/bin/python ocr_eval.py                          # EasyOCR 기준선
ENGINE=paddle ~/ocr-env/bin/python ocr_eval.py            # PaddleOCR (격리 환경 파이썬!)
IMAGES=images_pre ~/rnd-env/bin/python ocr_eval.py        # 전처리본 채점
```
- ⚠️ ENGINE과 파이썬을 짝 맞출 것: easy=rnd-env / paddle=ocr-env (반대면 ImportError).

## 3) OpenCV 전처리 복구

```bash
~/rnd-env/bin/python preprocess.py        # images/ → images_pre/ (열화별 복구 4종 + clean 대조군)
```

## 4) 돋보기 (신뢰도·박스 원자료 — 원인 조사)

```bash
~/rnd-env/bin/python ocr_easy.py                          # 기본 clean.png
IMAGE=images/rotated.png ~/rnd-env/bin/python ocr_easy.py
```

## 5) 미니 파이프라인 (② OCR → ③ 스팸·NER·PII)

```bash
~/rnd-env/bin/python ocr_pipeline.py      # 텍스트 소스 3종 × 하류 3종 매트릭스 출력
```
- 전제: 스팸 3종 세트(rnd-dataset-artifacts)·ner_klue.pt(rnd-detection-models)·ko-pii 설치.
- OCR은 내부에서 엔진별 환경 서브프로세스로 실행 — 인계 파일은 out/에 남음.

## 재현 주의

- 생성물(images*/·out/)은 git 미추적 — 스크립트로 재생성.
- 채점은 CPU 고정(잣대 안정성). 속도가 필요한 대량 추론에서만 GPU 검토(CPU 대조 검증 후).
