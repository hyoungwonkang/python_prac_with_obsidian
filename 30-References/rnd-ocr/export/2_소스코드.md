# 2. 소스코드 — OCR 추출 계층

> 실행법은 [[3_사용법]], 설계·결과는 [[1_연구문서]].

## 파일 5종 (실험 순서대로)

| 파일 | 역할 | 핵심 |
|---|---|---|
| `make_test_images.py` | 조건 통제 테스트 이미지 생성 | 정답 아는 문장을 PIL로 렌더링(clean) → 열화 4종 파생(회전·노이즈·저대비·축소). seed 고정 = 재현성 |
| `ocr_easy.py` | **돋보기** — EasyOCR 원자료 | `readtext()` 3요소(박스·텍스트·신뢰도) 그대로 표시. 원인 조사·신뢰도 확인용 |
| `ocr_eval.py` | **잣대** — 엔진 중립 CER 채점 | `edit_distance()`(레벤슈타인 DP) + `show_diff()`(오독 구간) + `make_reader()`(ENGINE=easy\|paddle 같은 인터페이스로 래핑) |
| `preprocess.py` | OpenCV 전처리 복구 4종 | deskew(Otsu→minAreaRect 각도→역회전)·denoise(NL-Means)·binarize(Otsu)·upscale(INTER_CUBIC). 열화와 1:1 짝 |
| `ocr_pipeline.py` | ②→③ 미니 파이프라인 | 텍스트 소스 3종(정답/Easy/Paddle) × 하류 3종(스팸·NER·PII). 하류는 기존 산출물 재사용 |

## 핵심 설계 4가지

1. **돋보기/잣대 분리** — 채점(정답 대조·객관)과 원인 조사(신뢰도·주관)를 다른 파일로.
   잣대는 신뢰도를 안 씀: 모델의 자기 확신은 채점 근거가 될 수 없다.
2. **엔진 중립 잣대** — `make_reader()`가 엔진을 같은 인터페이스로 감싸 채점부는 엔진을 모름.
   실행당 한 엔진만 import (torch↔paddle 동시 로드 크래시도 구조적으로 회피).
3. **전처리는 잣대 재사용으로 검증** — 복구본을 images_pre/에 쓰고 `IMAGES=images_pre`로
   같은 잣대 재채점. 채점기 변경 0줄 = "전처리 덕분"임을 보장.
4. **파이프라인은 서브프로세스 + 파일 인계** — 엔진별 파이썬이 다름(easy=rnd-env,
   paddle=ocr-env)을 계층 분리로 활용: ②는 별도 프로세스, out/*.txt로 ③에 인계.
   `ocr_eval.make_reader`를 import 재사용(코드 중복 0). 하류 로딩은 기존 스크립트 패턴
   (classify_text.py 3종 세트 로딩·predict_ner.py 함수 import·ko-pii process()).

## deskew 알고리즘 (preprocess.py:26-40)

```
① Otsu 이진화(반전) → 글자 픽셀만 흰색으로
② cv2.findNonZero → 글자 픽셀 좌표 전부 수집
③ cv2.minAreaRect → 픽셀 무리를 감싸는 최소 회전 사각형의 각도 = 문서 기울기
   (실측: 8° 회전 이미지에서 -7.996° 검출 — 내용 이해 없이 기하만으로 진단)
④ warpAffine 역회전 (흰 배경 채움)
```
